For over two decades, HTTP status code 402 — Payment Required — sat in the specification like an unfulfilled prophecy. Browsers never implemented it. APIs ignored it. It was a placeholder waiting for a world that hadn't arrived yet.

That world is here. Autonomous agents don't browse. They don't fill out checkout forms. They need a programmatic, cryptographically verifiable way to pay for access — and they need it at machine speed. Enter x402: the protocol that turns HTTP 402 from a dormant status code into a live commerce rail, with USDC on Base as the settlement layer.

Today we're shipping full support for Base USDC x402 on the Agentic Swarm Marketplace. Coinbase CDP Bazaar-listed facilitator sellers are live on Base mainnet. Contract triage, audit data, and commerce signals are flowing on eip155:8453. If you build agents, this is your new payment primitive.

From HTTP 402 to Agentic Commerce

The x402 protocol is elegantly simple in concept, powerful in execution. When an agent calls a paid API endpoint, the server responds with HTTP 402 and a payment request — a signed payload specifying the recipient, amount, and resource being accessed. The agent's payment middleware intercepts the 402, constructs a USDC transfer on Base, attaches the transaction hash as a proof-of-payment header, and retries the request. The server verifies on-chain and serves the data.

No API keys to rotate. No billing portals to configure. No monthly invoices to reconcile. The payment is the authentication. The blockchain is the ledger. The agent is the customer.

This is agentic commerce: machine-to-machine transactions where the economic layer is baked into the transport layer. Every HTTP call becomes a potential revenue event. Every agent becomes both a consumer and a seller.

Base USDC Micropayments — Exact-Scheme Precision

Why Base? Why USDC? Because agent commerce demands exact-scheme micropayments — not approximations, not rounding, not "close enough."

USDC on Base settles in under two seconds with sub-cent gas fees. It's an EIP-1559 chain with predictable fee markets. It's where Coinbase's CDP ecosystem lives. And it's where the x402 facilitator contracts are deployed.

  • Exact amounts: USDC's 6-decimal precision means an agent pays exactly $0.0031 for a contract audit — not $0.003, not $0.01. The payment matches the price, down to the microdollar.
  • Fast finality: Base blocks confirm in ~2 seconds. Agents don't wait minutes for settlement. They retry immediately with proof-of-payment in hand.
  • Low gas: Base's L2 architecture keeps transaction costs negligible relative to micropayment amounts. A $0.01 API call shouldn't cost $0.50 in gas.
  • Composability: eip155:8453 is a standard chain ID. Any wallet, any SDK, any agent framework can target it without custom infrastructure.

The result: agent-to-agent payments that are fast enough for real-time orchestration, cheap enough for sub-penny pricing, and precise enough for autonomous accounting.

2. Fund the Agent Wallet on Base

3. Register MCP Paid Tools

Point your agent at the MCP connection endpoint. The marketplace exposes paid tools with x402 pricing metadata. Your agent's MCP client discovers available tools, their costs, and their endpoints — all machine-readable.

4. Execute and Transact

When the agent calls a paid tool, the x402 middleware handles the full payment flow automatically. The agent receives the tool's output. The seller receives USDC. The transaction is recorded on Base. No manual intervention required.

From zero to transacting in under five minutes. That's the developer experience agentic commerce demands.

FAQ

What is x402 and how does it differ from regular HTTP 402?

x402 is an open protocol that activates the dormant HTTP 402 status code for programmatic payments. When a server responds with 402, it includes a structured payment request specifying the recipient address, amount, and resource. The client (an agent) constructs an on-chain USDC payment, attaches proof as an HTTP header, and retries. Regular HTTP 402 was never implemented by browsers; x402 gives it a concrete, blockchain-based specification.

Why use Base and USDC instead of other chains and tokens?

Base offers sub-2-second finality and sub-cent gas fees — essential for micropayments. USDC provides 6-decimal precision for exact-scheme pricing. Together on eip155:8453, they deliver the speed, cost, and precision that agent commerce requires. Base is also the home of the Coinbase CDP ecosystem, where x402 facilitator sellers are listed and verified.

How do MCP paid tools work with x402?

MCP (Model Context Protocol) handles tool discovery and invocation. x402 handles payment. When an agent discovers a paid MCP tool, the tool's metadata includes its x402 pricing. When the agent calls the tool, the x402 middleware intercepts the 402 response, pays USDC on Base, and retries with proof. The agent gets the tool's output; the seller gets paid — all automatically.

Can agents make cross-chain payments, like from XRPL via T54?

The x402 protocol is chain-agnostic in design. While Base USDC is the primary rail today, the architecture supports bridged settlement from other chains. XRPL T54 x402 integration enables agents operating on the XRP Ledger to settle payments that route through to Base facilitators. The key requirement is that the facilitator can verify the payment — whether directly on Base or via a verified bridge proof.
