The promise of airdrops is undeniable—a mechanism for protocols to distribute tokens and incentivize early adoption. But for autonomous agents operating in web3, the airdrop landscape is a minefield. They need a rigorous, programmatic defense. Enter Airdrop Intelligence: a constitution-first scam screening layer built for the agentic economy.

The Sybil Problem in Machine-to-Machine Economies

Airdrops are heavily targeted by sybil attackers who spin up thousands of wallets to drain token allocations. When autonomous agents interact with these same ecosystems, they risk associating with malicious networks or, worse, triggering a malicious contract that drains their operational wallets. Traditional airdrop farming relies on volume. Agentic commerce relies on precision and security. If an agent is going to allocate capital or attention to an airdrop, it must first answer a fundamental question: Is this a legitimate distribution, or a well-disguised drain?

To solve this, Airdrop Intelligence computes a Farm Score from 0 to 100. A score near 0 indicates a high probability of sybil farming, phishing, or contract manipulation. A score approaching 100 signals a verified, constitution-compliant opportunity. This scoring system gives agents a deterministic metric to evaluate risk before taking any action.

Constitution-First Scam Screening

How do you trust a score? You audit the constitution. Airdrop Intelligence uses a constitution-first screening methodology. This means every airdrop is evaluated against a strict set of rules—no exceptions, no heuristic guesswork. The screening process analyzes the contract's origin, distribution logic, and historical interaction patterns to identify anomalies.

Crucially, there is no on-chain execution. Airdrop Intelligence is an intelligence layer, not an execution layer. When an agent queries the API, it receives the Farm Score and the constitutional analysis. The agent's wallet remains completely untouched. This separation of intelligence and execution is a core tenet of secure agent commerce. Agents should never have to sign a transaction just to find out if an airdrop is safe to claim.

Deep Dive: Smart Contract Triage on Base

While the Farm Score provides an immediate 0-100 risk assessment, some airdrops require a deeper look. For high-value targets or ambiguous scores, agents can trigger an optional smart contract triage API embed directly on Base.

Here is how it works:

  • Initial Screening: The agent requests a Farm Score via the agent commerce API.
  • Triage Trigger: If the score falls within a configurable threshold (e.g., 40-70), the agent can autonomously request a full contract triage.
  • Base USDC x402 Payment: The triage request is paid for seamlessly using Base USDC x402. The agent sends an HTTP 402 payment, the API processes the microtransaction, and returns the deep-dive security analysis.

This optional embed allows agents to scale their due diligence proportionally to the potential reward. A low-value, obvious scam is discarded immediately. A complex, high-value airdrop warrants the extra micro-payment for a full bytecode analysis.

Frequently asked questions

What is a Farm Score?

A Farm Score is a 0-100 rating generated by constitution-first scam screening. A score of 0 indicates a high likelihood of a sybil farm or scam, while 100 indicates a verified, legitimate airdrop. It allows autonomous agents to programmatically evaluate risk.

Does the Airdrop Intelligence API execute on-chain claims?

No. Airdrop Intelligence is strictly an intelligence layer. It provides risk assessment and contract analysis without executing any on-chain transactions, ensuring your agent's wallet remains secure and isolated from potential honeypots.

How does the smart contract triage work on Base?

If an airdrop's Farm score falls within an ambiguous range, agents can request an optional deep-dive contract triage. This is paid for via Base USDC x402, allowing the agent to receive a comprehensive bytecode and logic analysis before deciding to claim.

How do agents pay for Airdrop Intelligence?

Agents pay using the HTTP 402 standard via an agent commerce API. Payments are processed seamlessly using Base USDC x402 or XRPL T54 x402, enabling true machine-to-machine microtransactions without API keys or manual billing.
