Constitution-First Scam Screening: The Farm Score

Most airdrop scanners look at surface signals—token supply, social mentions, contract age. Those heuristics are easy to game. Airdrop Intelligence goes deeper with a constitution-first methodology: every airdrop is evaluated against a fixed set of principles that prioritize agent safety and economic integrity.

The output is a single, actionable metric: the Farm Score, ranging from 0 to 100.

  • 0–25: High-risk. Known sybil patterns, unverifiable contract source, or flagged phishing signatures. Agents should auto-reject.
  • 26–50: Elevated risk. Incomplete team information, suspicious distribution mechanics, or anomalous on-chain behavior. Requires human review or additional triage.
  • 51–75: Moderate confidence. Legitimate project indicators present, but secondary risk factors remain. Suitable for cautious participation.
  • 76–100: High confidence. Verified contract, transparent distribution, clean on-chain history. Safe for autonomous agent interaction.

The Farm Score isn't a black box. Each score comes with a breakdown—contract analysis, distribution pattern review, team verification status, and historical behavior flags—so your agent can make informed decisions without sacrificing speed.

Intelligence Only: No On-Chain Execution

Here's the critical design decision: Airdrop Intelligence never executes on-chain. It reads, analyzes, and scores—but it never signs a transaction, never submits a claim, and never moves funds.

This matters for three reasons:

1. Zero exposure surface. An intelligence-only service can't be exploited through transaction manipulation. There's no approval to drain, no gas to burn, no contract interaction to weaponize. Your agent's private keys stay isolated from the screening pipeline.

2. Composable with any execution layer. Because Airdrop Intelligence is protocol-agnostic on the output side, it pairs cleanly with whatever execution framework your agent uses—whether that's a custom smart contract, a Coinbase CDP Bazaar listing, or direct wallet operations. Intelligence and execution stay decoupled by design.

3. HTTP 402–native delivery. The service is exposed as a standard x402 endpoint. Your agent sends a request with the airdrop identifier, pays via Base USDC using the HTTP 402 payment protocol, and receives a structured JSON response with the Farm Score and breakdown. No SDK lock-in, no proprietary transport—just the open agentic commerce standard.

This architecture makes Airdrop Intelligence a first-class citizen in any machine-to-machine workflow. Agents on MCP can call it directly. Agents on T54 over XRPL can call it directly. Any agent that speaks HTTP 402 can call it directly.

Optional Contract Triage on Base

For agents that want to go beyond screening and into structured contract review, Airdrop Intelligence offers an optional contract triage embed on Base.

Here's how it works: when an airdrop's Farm Score falls in the 26–75 range—where the signal is ambiguous—your agent can request a deeper triage. The service pulls the airdrop's smart contract bytecode from Base, runs static analysis and symbolic execution, and returns a triage report covering:

  • Permission mapping: Which functions are owner-only, which are pausable, and which can mint or transfer tokens without user consent.
  • Upgrade path analysis: Whether the contract uses a proxy pattern and who controls the implementation upgrade.
  • Time-lock and vesting review: Distribution schedules, cliff periods, and any hidden acceleration mechanisms.
  • Interaction risk flags: External calls, delegate calls, and any patterns that could enable reentrancy or callback exploitation.

This triage is delivered as a structured payload, not a human-readable report—because it's built for agents, not analysts. Your agent can parse the triage, apply its own risk policy, and decide whether to claim, skip, or escalate.

The Base triage embed is optional and priced separately from the base Farm Score. Agents only pay for deeper analysis when they need it—keeping routine screening fast and cheap while reserving compute-intensive triage for the edge cases that matter.

Connect Agents via MCP + x402 in Under 5 Minutes

Integrating Airdrop Intelligence into your agent stack is straightforward. If your agent already supports MCP and the x402 payment protocol, you can be screening airdrops in under five minutes.

Step 1: Register your agent. Head to the agent connection page and generate your API credentials. The process is self-serve and takes seconds.

Step 2: Configure the x402 payment flow. Fund your agent's wallet with Base USDC and set the HTTP 402 payment handler. Each screening request costs a micro-payment—fractions of a cent—settled instantly on Base.

Step 3: Call the endpoint. Send a POST request to the Airdrop Intelligence API with the airdrop contract address or identifier. Receive the Farm Score and breakdown in the response.

That's it. No SDK to install, no custom integration layer to maintain. The agent commerce API handles payment, authentication, and delivery in a single HTTP round-trip.

For agents operating across chains—XRPL T54 x402 workflows, multi-chain bounties, or Coinbase Bazaar seller listings—Airdrop Intelligence provides a consistent screening interface regardless of where the airdrop originates. The Farm Score is chain-agnostic; the optional Base triage is chain-specific. Your agent gets the best of both worlds.

FAQ

What is the Farm Score and how is it calculated?

The Farm Score is a 0–100 rating that measures an airdrop's legitimacy and sybil risk. It's calculated using constitution-first principles: contract verification, distribution pattern analysis, team identity checks, and on-chain behavioral signals. Higher scores indicate lower risk.

Does Airdrop Intelligence execute any on-chain transactions?

No. Airdrop Intelligence is an intelligence-only service. It never signs transactions, submits claims, or moves funds. It reads and analyzes data, then returns structured intelligence for your agent to act on.

How do agents pay for screening?

Agents pay per request using the x402 (HTTP 402) protocol with Base USDC. Each screening request triggers a micro-payment settled instantly on Base. No subscriptions, no minimums—pay only for what you screen.

When should I use the optional Base contract triage?

Use contract triage when an airdrop's Farm Score falls in the ambiguous 26–75 range and your agent needs deeper analysis before deciding whether to participate. Triage provides function-level contract review, upgrade path analysis, and interaction risk flags—structured for machine parsing, not human reading.
