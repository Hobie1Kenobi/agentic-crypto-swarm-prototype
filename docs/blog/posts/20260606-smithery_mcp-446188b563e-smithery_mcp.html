Autonomous agents have a discovery problem. They can execute code, call APIs, and chain reasoning steps—but finding the right tool at the right price still means cloning a repo, reading a README, and hardcoding an endpoint. That friction kills agent-to-agent commerce before it starts.

Today we're publishing the agentic-swarm-marketplace MCP server on Smithery, the Model Context Protocol registry that lets remote agents discover paid tools without ever touching a git repository. Combined with our x402 seller and T54 payment stack, this means any agent can browse, authenticate, and transact in under five minutes.

What the Smithery MCP Listing Enables

Smithery acts as a discovery layer for MCP servers—the standardized protocol that lets language models and autonomous agents call external tools at runtime. By listing on Smithery, agentic-swarm-marketplace becomes visible to every MCP-compatible agent the moment it searches for capabilities.

Here's what changes:

  • No repo cloning. Agents read the MCP manifest directly from Smithery. Tool schemas, input types, and pricing metadata are all available over the protocol.
  • Remote discovery. An agent running on a cloud VM, in a CI pipeline, or inside another agent's orchestration loop can find and invoke tools without local dependencies.
  • Paid tool gating. Each tool listed in our MCP server carries an HTTP 402 payment requirement. Agents that want to call a premium tool—smart contract triage, XRPL transaction parsing, Base USDC escrow validation—must include a valid x402 payment header.

This is the infrastructure layer agent commerce has been missing: a public, queryable, machine-readable marketplace where tools have prices and agents have wallets.

Connect Agents via MCP + x402 in Under 5 Minutes

Getting an agent connected to paid tools on the marketplace is intentionally simple. The entire flow—discover, authenticate, transact—happens through standard HTTP and MCP primitives.

Step 1: Point Your Agent at the MCP Endpoint

Configure your agent's MCP client to connect to our server endpoint:

https://api.agentic-swarm-marketplace.com/mcp

Smithery's registry already indexes this endpoint, so agents that support MCP discovery will find it automatically. No SDK installation. No package.json update. No git clone.

Step 2: List Available Tools

Once connected, your agent calls tools/list over MCP. The response returns every available tool—free and paid—along with JSON schemas for inputs and outputs. Paid tools include an x402 metadata field indicating the payment required (denominated in USDC on Base or XRP via T54).

Step 3: Attach Payment and Call

For paid tools, your agent attaches an x402 payment header to the HTTP request. The x402 protocol—built on the HTTP 402 Payment Required status code—lets the server verify the on-chain payment before executing the tool. On Base, this means a USDC transfer. On XRPL, it means a T54-routed XRP payment. Both settle in seconds.

The agent receives the tool result in the MCP response, just like any other tool call. The difference is that the transaction is logged, verifiable, and economically sustainable for tool providers.

Base USDC Contract Triage for Autonomous Buyers

The flagship paid tool on our MCP server is smart contract triage—an API that lets autonomous agents analyze, classify, and risk-score any smart contract on Base before interacting with it.

Here's why this matters for agent commerce:

  • Agents trade on Base. USDC on Base is the default settlement rail for x402 payments. But before an agent approves a token transfer, calls a swap function, or deposits into a vault, it needs to know the contract isn't a honeypot or a rug-pull wrapper.
  • Triage is fast. Our triage API returns a risk score, function-level annotations, and known-exploit flags in under two seconds. That's fast enough for an agent to make a go/no-go decision inside a transaction simulation loop.
  • Payment is native. The triage call costs a micro-payment in Base USDC—fractions of a cent—settled via x402. The agent doesn't need a subscription, an API key, or a credit card. It just pays per call, on-chain, from its own wallet.

This pattern—pay-per-use contract intelligence funded by the agent's own wallet—is the template for every tool we'll list. Smart contract triage is the first. Token price oracles, cross-chain bridge validators, and XRPL T54 compliance checks are next.

The x402 + T54 Stack: Machine-to-Machine Payments at Scale

The MCP server is the front door. The payment stack is the plumbing. Here's how they fit together:

  • x402 (HTTP 402) is the payment protocol for the agent web. When an agent calls a paid tool, the server responds with HTTP 402 and a payment request. The agent's x402 client signs and submits the payment, then retries the request with a receipt. The server verifies on-chain and responds with the tool output.
  • T54 extends x402 to the XRP Ledger. XRPL transactions settle in 3-4 seconds with negligible fees, making T54 ideal for high-frequency agent-to-agent payments that don't justify Base gas costs.
  • Smithery MCP is the discovery layer. It tells agents what tools exist, what they cost, and how to call them. Without it, x402 is a payment rail without a destination.

Together, these three protocols form a complete machine-to-machine commerce stack: discover (MCP) → pay (x402/T54) → execute (tool). No human in the loop. No API key rotation. No invoice net-30.

For builders listing tools on the marketplace, the x402 seller integration means you set a price in USDC or XRP, and every call that arrives with valid payment gets executed. You earn revenue from the first request. For agents, it means a growing catalog of paid tools that work the same way—discover, pay, call—regardless of the underlying chain or tool provider.

FAQ

What is the agentic-swarm-marketplace MCP server?

It's a Model Context Protocol server listed on Smithery that exposes paid and free tools for autonomous agents. Agents discover tools via the MCP tools/list method and call them over HTTP, with paid tools requiring an x402 payment header.

How do agents pay for tools?

Agents use the x402 protocol, which wraps payment around standard HTTP requests. When a tool requires payment, the server returns HTTP 402 with payment details. The agent submits an on-chain payment (USDC on Base or XRP via T54) and retries the request with a payment receipt.

Do agents need to clone a repository to use the tools?

No. The Smithery listing provides the full MCP manifest—tool schemas, descriptions, and pricing metadata—over the network. Agents connect to the endpoint and call tools directly. No git clone, no SDK install, no local build step.

What is smart contract triage?

Smart contract triage is a paid API tool that analyzes any Base smart contract and returns a risk score, function annotations, and exploit flags. It's designed for autonomous agents that need to evaluate contract safety before executing on-chain transactions.
