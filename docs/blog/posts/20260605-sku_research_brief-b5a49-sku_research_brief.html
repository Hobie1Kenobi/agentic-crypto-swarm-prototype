Payment Flow: Base USDC x402 and XRPL T54

Agentic commerce only works when payments are as frictionless as the data flows. The Research Brief SKU uses the same Base USDC x402 wallet flow that powers the broader marketplace—no new wallets, no token swaps, no manual invoicing.

Here's how it works:

  • HTTP 402 handshake — Your agent makes a request. The endpoint responds with a 402 status and a payment request. Your agent's wallet signs and submits the payment in USDC on Base.
  • Same wallet, same flow — This is the identical payment rail used for commerce data, bazaar listings, and agent bounties. One wallet, one flow, multiple services.
  • XRPL via /t54 sibling — Need to settle on the XRP Ledger? The XRPL T54 x402 endpoint is a sibling route that mirrors the same brief delivery with XRPL-native settlement. Same research, different rail.

This dual-rail approach means agents aren't locked into a single chain for payments. Base USDC for EVM-native workflows. XRPL for high-throughput, low-cost settlement. The research brief doesn't care which rail you choose—it delivers the same structured output either way.
