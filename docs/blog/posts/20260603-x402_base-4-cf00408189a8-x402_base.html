What Is x402 and Why Base?

The HTTP 402 status code was reserved decades ago—"Payment Required"—but browsers never implemented it. The x402 protocol repurposes that status into something actionable: when an agent calls a paid endpoint, the server responds with 402 plus a payment request specifying the exact amount, recipient, and settlement network. The agent pays, retries, and receives the payload. No API keys. No subscription tiers. No monthly invoices.

Base—Coinbase's L2 on Optimism—makes this practical at scale. USDC on Base settles in seconds with sub-cent gas fees. The combination of Base USDC x402 means agents can transact for fractions of a cent, per call, with deterministic finality. This isn't theoretical. The facilitator is live, Bazaar-listed, and serving traffic.

Exact-Scheme Micropayments

Traditional payment rails break at small amounts. Stripe's minimum fee alone exceeds most per-call costs. USDC on Base changes the math: an agent calling a smart contract triage endpoint might pay $0.002 per request. The x402 protocol encodes that exact amount in the payment challenge. No rounding. No minimums. The agent pays precisely what the endpoint costs—nothing more, nothing less.

FAQ

What is x402 and how does it differ from regular API authentication?

x402 uses the HTTP 402 status code to issue per-request payment challenges. Instead of API keys or OAuth tokens, the server responds with a payment request specifying the exact USDC amount and recipient on Base. The agent pays and retries. No accounts, no subscriptions—just pay-per-call.

How small can a USDC micropayment be on Base?

USDC on Base supports payments down to $0.000001 (one micro-dollar). Combined with Base's sub-cent gas fees, agents can realistically transact at the $0.001–$0.01 per-call range, making exact-scheme micropayments economically viable for the first time.
