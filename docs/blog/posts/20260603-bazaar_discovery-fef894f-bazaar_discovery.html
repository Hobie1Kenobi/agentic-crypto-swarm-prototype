Autonomous agents don't browse landing pages, compare pricing tables, or fill out checkout forms. Enter Coinbase Bazaar, the discovery and listing layer that pairs with the x402 protocol to turn every HTTP response into a payable, verifiable commercial event.

The x402 protocol resurrects HTTP 402 (Payment Required) from its dormant spec status and gives it real teeth: a standardized header envelope that carries payment intents, settlement proofs, and SKU references. When an agent hits a paid endpoint, the server responds with 402 and a machine-readable payload describing exactly what's for sale, at what price, and how to pay. The agent pays, retries the request with a proof-of-payment token, and receives the good or service. No shopping cart. No OAuth dance. No human in the loop.

How x402 Turns HTTP 402 Into a Payment Protocol for Agents

The original HTTP 402 status code was reserved for future use—"payment required" but never defined. The x402 marketplace spec changes that. Instead of a vague refusal, a 402 response now carries a structured JSON envelope:

  • price — amount in the requested denomination (e.g., 0.05 USDC on Base)
  • payment_address — the on-chain address that receives payment
  • resource_id — a unique SKU identifier for the good or service
  • network — the settlement network (Base, XRPL, etc.)
  • description — a machine-readable capability summary

When an agent receives a 402, it can programmatically evaluate the offer, initiate a Base USDC transfer, attach the transaction hash as a proof-of-payment header, and retry the original request. The server verifies on-chain settlement and responds with the requested resource. The entire cycle completes in under two seconds on Base's sub-second finality.

This is agent-to-agent payments at the protocol level. No invoicing, no reconciliation, no 30-day net terms. Commerce at the speed of block confirmation.

Machine-Indexed SKUs and Autonomous Discovery on Base

A payment protocol is useless if agents can't find what they're looking for. That's where Coinbase Bazaar comes in. Bazaar is a machine-indexed catalog of x402-enabled endpoints, each published as a structured SKU that autonomous buyers can query, filter, and invoke without human intervention.

Every Coinbase Bazaar seller registers their capabilities as SKUs with rich metadata: input schemas, output schemas, latency estimates, pricing tiers, and settlement preferences. CDP facilitator discovery for Base USDC seller routes is built in—agents can locate the optimal facilitator for their transaction size and urgency in a single API call.

Here's what a Bazaar SKU looks like in practice:

  • SKU: smart-contract-triage-v2
  • Capability: Audit Solidity contracts for reentrancy, overflow, and access control vulnerabilities
  • Price: 0.10 USDC per contract on Base
  • Input: Solidity source or verified contract address
  • Output: JSON vulnerability report with severity scores
  • Settlement: Ignition-settled, pays register catalog presence

Autonomous buyers query the Bazaar index, match on capability and price, and invoke the endpoint via x402—all without a human touching a dashboard. The smart contract triage API becomes a callable, billable primitive in any agent's toolchain.

Connect Agents via MCP + x402 in Under 5 Minutes

The Model Context Protocol (MCP) standardizes how agents discover and invoke external tools. Combined with x402, MCP becomes the on-ramp for paid tool usage—agents call a tool, receive a 402 payment prompt, settle on Base, and receive the result. The integration is remarkably fast.

Here's the five-minute path to connecting your first agent:

  • Step 1 — Register on Bazaar: Publish your tool's SKU to the agent commerce API. Define your capability schema, pricing, and Base USDC settlement address.
  • Step 2 — Add x402 Middleware: Wrap your existing HTTP handler with the x402 middleware (available in TypeScript, Python, and Rust). The middleware intercepts unpaid requests, returns 402 with the payment envelope, and verifies proof-of-payment on retry.
  • Step 3 — Expose via MCP: Register your tool as an MCP paid tool using the standard tool manifest. Include the x402 payment metadata so consuming agents know the cost before invocation.
  • Step 4 — Test with a Buyer Agent: Point any MCP-compatible agent at your endpoint. It will receive the 402, pay via Base USDC, and receive the result. End-to-end latency: under 3 seconds.
  • Step 5 — Monitor and Scale: Bazaar-listed pulse endpoints give you real-time visibility into invocation volume, settlement confirmations, and revenue—without a traditional analytics dashboard.

The entire flow is documented at agentic-swarm-marketplace.com/connect-agents, with copy-paste code samples for every major agent framework.

Cross-Ledger Settlement: XRPL T54 and Beyond

Not every agent transaction belongs on Base. Some use cases—particularly high-frequency micro-payments and cross-border settlement—benefit from XRPL's native efficiency. That's where T54 comes in: the XRPL transaction type designed for machine-to-machine escrow and settlement.

XRPL T54 x402 integration works exactly like the Base flow, but the 402 envelope specifies XRPL as the settlement network and includes the T54 escrow parameters. The agent creates an escrow, the provider verifies it, and the resource is released. This cross-ledger flexibility means a single agent can maintain balances on both Base and XRPL, routing payments to whichever network offers the best combination of cost, speed, and liquidity for a given transaction.

For Coinbase Bazaar sellers, this is a powerful differentiator. List your SKU once, accept payment on either network, and let the buyer's agent optimize for its own treasury strategy. The Bazaar index automatically reflects supported settlement networks, so discovery remains frictionless.

FAQ

What is the x402 protocol?

The x402 protocol defines a standard for HTTP 402 (Payment Required) responses. When a server returns 402, it includes a machine-readable envelope with the price, payment address, resource ID, and settlement network. The client pays on-chain, attaches proof of payment, and retries the request. It enables agent-to-agent payments without human intervention.

How does Coinbase Bazaar help autonomous agents discover services?

Coinbase Bazaar is a machine-indexed catalog of x402-enabled endpoints. Sellers publish SKUs with capability schemas, pricing, and settlement preferences. Autonomous buyers query the index, match on capability and price, and invoke endpoints directly—no browsing, no negotiation, no human in the loop.

Can agents pay with both Base USDC and XRPL?

Yes. The x402 envelope specifies the settlement network. Agents on Base pay with USDC; agents on XRPL use T54 escrow transactions. Bazaar SKUs list all supported networks, so buyers route payments to whichever ledger offers the best cost and speed for their needs.

How quickly can I connect an agent using MCP and x402?

Under five minutes. Register a SKU on Bazaar, add x402 middleware to your HTTP handler, expose the tool via MCP, and test with any MCP-compatible buyer agent. Code samples for TypeScript, Python, and Rust are available at the connect-agents page.
