The Problem: Agents Need Citable Research, Not Just Completions

Large language models generate fluent text, but fluency isn't the same as rigor. An agent performing due diligence on a DeFi protocol can't afford hallucinated statistics or uncited claims. Traditional LLM outputs lack the structure that downstream verification workflows demand.

Consider what an autonomous buyer actually requires:

  • Sectioned analysis — separate treatments of tokenomics, team history, on-chain activity, and risk factors, not a single undifferentiated wall of text.
  • In-line citations — every factual claim linked to a source the agent can re-verify programmatically.
  • Discipline-specific formatting — whether the brief covers legal risk, technical architecture, or market positioning, the structure must reflect the domain.

Without these properties, an agent's downstream decisions rest on ungrounded prose. The Research Brief SKU solves this by wrapping a disciplined, citation-aware generation pipeline behind a single HTTP 402 endpoint.

How the Research Brief SKU Works

The Research Brief SKU is a machine-to-machine product on the x402 marketplace. An agent—whether operating independently or orchestrated through a broader workflow—sends a request to the endpoint and receives a structured, multi-section brief in return. Payment is handled automatically via the HTTP 402 protocol.

Here's the flow at a glance:

  • Request — The agent POSTs a target (contract address, protocol name, or entity identifier) to the research brief endpoint.
  • Payment — The endpoint responds with an HTTP 402 status and a payment request denominated in Base USDC. The agent's wallet authorizes the micro-payment using the same x402 flow that powers all commerce data on the platform.
  • Generation — Once payment confirms, the LLM pipeline produces a multi-section brief with in-line citations drawn from verified sources.
  • Delivery — The structured JSON payload returns to the agent, ready for ingestion into decision pipelines.

Because the payment rails are identical to those used for other data products on the platform, agents already configured for Base USDC x402 transactions can add the Research Brief SKU to their workflows with zero additional wallet configuration.

Citation Discipline

Every factual assertion in the brief is tagged with a source URL or on-chain reference. This isn't a post-hoc bibliography appended to the end of a document—it's inline attribution woven into each claim. Agents can programmatically verify citations, cross-reference them against their own data sources, and weight confidence scores accordingly.

This citation discipline is what transforms a research brief from a helpful summary into a due-diligence artifact—something an agent can log, audit, and chain into subsequent reasoning steps.

Dual-Rail Payments: Base USDC x402 and XRPL T54

The Research Brief SKU settles payments on Base USDC x402, the same rail that powers the broader agent commerce API. For agents operating on XRPL, the /t54 sibling endpoint offers an alternative settlement path via the XRPL T54 x402 bridge.

Why dual rails matter:

  • Base USDC offers EVM-native agents low-gas, fast-finality settlements ideal for Coinbase Bazaar sellers and Ethereum-ecosystem workflows.
  • XRPL T54 extends the same HTTP 402 payment experience to agents whose treasury and identity live on the XRP Ledger.

Both rails share the same wallet flow. An agent that has already funded its Base USDC wallet for smart contract triage API calls can purchase research briefs without any additional setup. The payment experience is unified: one HTTP 402 handshake, one signed transaction, one confirmed brief.

This architecture reflects a core principle of the agentic commerce layer—payment should be infrastructure, not a product feature. Agents shouldn't need to reason about settlement mechanics; they should simply request, pay, and receive.
