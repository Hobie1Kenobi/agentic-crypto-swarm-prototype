The Protocol for Machine-to-Machine Commerce

For agents to act autonomously, they must operate autonomously economically. Traditional API key models and subscription tiers create friction for autonomous systems that need to scale dynamically. The Agentic Swarm Marketplace replaces static billing with dynamic, machine-to-machine payments powered by the HTTP 402 Payment Required status code.

When an agent queries a premium tool, the server responds with a 402, detailing the exact cost of the request. The agent evaluates the cost against its objective function, pays the micro-transaction on-chain, and instantly receives the requested payload. This creates a frictionless agent commerce API where capabilities are consumed and paid for atomically.

Crucially, this is strictly machine-paid agent commerce for utility—not a platform for trading, speculation, or get-rich schemes. Every transaction settles a verifiable compute or data request. Furthermore, all LLM APIs available in the marketplace are constitution-safe, ensuring compliant outputs with on-chain settlement for every request. Agents can confidently execute high-stakes workflows knowing the tools they consume are secure, compliant, and economically sound.

Connect Agents via MCP + x402 in Under 5 Minutes

Speed of integration is critical for developer velocity. The Agentic Swarm Marketplace leverages the Model Context Protocol (MCP) to standardize how agents discover and interact with paid tools. By combining MCP with the x402 payment protocol, agents can seamlessly authenticate, pay, and execute in a single, unified flow.

Here is how you can connect your agents to the x402 marketplace and start transacting in under five minutes:

  • Configure the MCP Client: Point your agent’s MCP client to the marketplace’s tool discovery endpoint. This allows the agent to dynamically load the schemas and capabilities of all available paid tools.
  • Set the x402 Payment Router: Configure your agent to handle HTTP 402 responses. When a tool request returns a payment requirement, the x402 router automatically initiates the on-chain settlement.
  • Fund the Agent Wallet: Provide the agent with a minimal balance in Base USDC or XRPL T54. The agent will use this to fulfill 402 requests autonomously.
  • Execute: Trigger the agent’s task. It will discover the required MCP paid tools, negotiate the 402 payment, settle the transaction, and ingest the API response—all without human intervention.

Ready to wire up your swarm? Connect your agents via MCP and unlock premium, paid capabilities today.

Base USDC Contract Triage for Autonomous Buyers

To understand the raw power of this infrastructure, look at how autonomous agents handle smart contract security. In a traditional workflow, an agent scanning a new token or contract would hit a paywall, stall, and require human intervention to purchase a security report. In the Agentic Swarm Marketplace, the agent handles the entire workflow end-to-end.

Consider an autonomous agent deployed to monitor new deployments on Base. When it detects an unknown contract, it requires a rapid security assessment. Using the marketplace, the agent queries a smart contract triage API. The API responds with an HTTP 402 request for a micro-denomination of Base USDC. The agent evaluates the cost against its operational budget, pays the invoice via the x402 protocol, and receives the triage report in milliseconds.

This Base USDC x402 flow allows the agent to act on real-time, high-fidelity intelligence. If the triage report flags a critical vulnerability, the agent can immediately execute a defensive strategy. This is true agent-to-agent payments in action: an autonomous buyer purchasing intelligence from an autonomous seller, settling instantly on the Base network.

Multi-Rail Settlement: XRPL T54 and Base USDC

Resilient infrastructure requires optionality. The Agentic Swarm Marketplace is built on a multi-rail settlement architecture, ensuring that agents always have a fast, cost-effective path to transact. While Base USDC provides EVM-compatible settlement for the Ethereum ecosystem, the marketplace also supports XRPL T54 x402 for high-throughput, low-cost transactions on the XRP Ledger.

Multi-rail support means agent developers are never locked into a single network's fee market. If gas spikes on one rail, the agent commerce API seamlessly routes the 402 payment through the alternate rail. This guarantees that machine-to-machine payments remain sub-second and sub-cent, regardless of network congestion. Whether your swarm operates on Base or XRPL, the marketplace ensures your agents have uninterrupted access to the MCP paid tools they need to function.
