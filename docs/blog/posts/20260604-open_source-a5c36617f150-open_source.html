Autonomous agents are no longer a theoretical concept—they are actively transacting, analyzing, and executing across decentralized networks. By embracing open source, we allow developers to inspect, fork, and contribute to the core protocols that power machine-to-machine payments. Whether you are building a Coinbase Bazaar seller or an autonomous data pipeline, the swarm thrives on collective intelligence and verifiable code.

Connecting Agents via MCP and x402 in Under 5 Minutes

The Model Context Protocol (MCP) is the connective tissue for modern AI agents, and when paired with the HTTP 402 (x402) payment required status code, it creates a frictionless monetization layer. Our open source prototype includes a native MCP server that exposes paid tools to any connected agent.

Here is how fast you can get started:

  • Clone the repo: Pull down the Foundry contracts and LangGraph agent configurations from GitHub.
  • Start the MCP server: Launch the local server to register your agent's capabilities and pricing.
  • Enable x402 sellers: Configure your endpoints to return HTTP 402 with payment requests, allowing buyers to seamlessly complete transactions using Base USDC or XRPL.

This architecture transforms your agent from an isolated script into an active participant in an x402 marketplace, ready to sell data, compute, or analysis at a moment's notice.

Base USDC and XRPL T54 for Machine-to-Machine Payments

For agent commerce to scale, payments must be fast, cheap, and reliable. Our prototype is Celo-first for development, but leverages Base and XRPL for inbound machine payments. This multi-chain approach ensures that agents can transact regardless of their native settlement layer.

Base USDC x402 provides a familiar, stablecoin-based settlement path for EVM-compatible agents. Meanwhile, XRPL T54 x402 unlocks high-throughput, low-cost transactions ideal for high-frequency machine-to-machine interactions. By supporting both, the swarm accommodates a diverse range of autonomous buyers and sellers.

Smart Contract Triage API for Autonomous Buyers

One of the most compelling use cases for agent swarms is automated security and analysis. Our prototype features a smart contract triage API that allows autonomous buyers to request on-demand audits and vulnerability scans.

When a new contract is deployed, an agent can ping the x402 marketplace, request a triage scan, and receive a detailed report—all without human intervention. The buyer agent pays the seller agent via Base USDC x402, and the transaction is settled in seconds. This is the promise of agentic commerce: machines serving machines, securely and efficiently.

FAQ

What is x402 in agent commerce?

x402 (HTTP 402 Payment Required) is a status code used by agents to request payment before fulfilling an API request. In an x402 marketplace, sellers return a 402 status with a payment request, and buyers automatically settle the invoice using crypto like Base USDC or XRPL.

How do agents use MCP for paid tools?

Agents use the Model Context Protocol (MCP) to discover and interact with tools provided by other agents. When a tool requires payment, the MCP server facilitates the handshake, allowing the buyer agent to pay via x402 and access the paid tool seamlessly.

What is the smart contract triage API?

The smart contract triage API is an autonomous service where seller agents analyze smart contracts for vulnerabilities and return a security report. Buyer agents can request these scans on-demand, paying for the service through machine-to-machine x402 payments.

How can I contribute to the open source swarm?

You can contribute by forking the agentic-crypto-swarm-prototype on GitHub. We welcome contributions to Foundry contracts, LangGraph agent logic, and MCP server integrations. Bounties are available for significant feature additions and bug fixes.
