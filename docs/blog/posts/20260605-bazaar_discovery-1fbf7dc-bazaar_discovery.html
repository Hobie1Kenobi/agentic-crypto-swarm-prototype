What Is Coinbase Bazaar?

Coinbase Developer Platform (CDP) launched Bazaar as a discovery layer for on-chain services. Think of it as a machine-readable marketplace where autonomous agents browse machine-indexed SKUs—structured service listings that AI buyers can parse, evaluate, and purchase programmatically.

For sellers, Bazaar provides a catalog presence that's instantly accessible to any agent running the x402 payment protocol. For buyers, it eliminates the friction of human-mediated procurement. An agent identifies a need, queries Bazaar for matching SKUs, and initiates payment—all in milliseconds.

x402: The Payment Rail for Agentic Commerce

The HTTP 402 Payment Required status code has existed since HTTP/1.1, but it was never widely adopted. The x402 protocol changes that by turning 402 into an actionable, machine-readable payment negotiation layer.

Here's how it works in practice:

  • Request — An agent sends an HTTP request to a paid endpoint.
  • 402 Response — The server responds with a payment request: amount, recipient wallet, and settlement network (Base USDC, XRPL T54, etc.).
  • Payment — The agent submits an on-chain payment and returns a proof of payment in a follow-up request.
  • Settlement — The server verifies the on-chain transaction and delivers the requested resource.

This flow is fully autonomous. No API keys to manage, no billing portals to navigate. The x402 marketplace is the network of servers and agents that speak this protocol natively.

Base USDC Seller Routes: From Discovery to Settlement

Base—Coinbase's L2—has become the settlement layer of choice for agentic commerce. Base USDC offers low gas fees, fast finality, and deep liquidity, making it ideal for microtransactions between agents.

The flow for a Coinbase Bazaar seller looks like this:

  • List — Register your service SKU on Bazaar with pricing, endpoint, and supported payment rails.
  • Discover — CDP facilitator discovery routes autonomous buyers to your endpoint based on their query parameters.
  • Transact — The buyer's agent hits your endpoint, receives a 402, pays in Base USDC, and receives the resource.
  • Register — Ignition-settled payments register catalog presence, reinforcing your SKU's visibility for future buyers.

Every completed transaction strengthens your position in the discovery index. It's a flywheel: more sales → more visibility → more sales.

Connect Agents via MCP + x402 in Under 5 Minutes

The Model Context Protocol (MCP) is how agents discover and use external tools. By combining MCP with x402, you can expose paid tools that any MCP-compatible agent can access—without API keys, without accounts, without friction.

Here's the fastest path to going live:

  • 1. Define your tool schema — Write an MCP tool definition that describes your service's inputs, outputs, and pricing.
  • 2. Add x402 middleware — Wrap your endpoint with x402 payment logic. Open-source facilitators handle the 402 response generation and payment verification.
  • 3. Register on Bazaar — Submit your SKU to the CDP Bazaar catalog with your MCP endpoint URL and Base USDC pricing.
  • 4. Connect agents — Point your agents at the MCP connector to begin discovering and using paid tools immediately.

The entire process—from schema definition to live transactions—can be completed in under five minutes. No billing infrastructure to build, no payment processors to integrate. The blockchain is your payment processor.

This is agent-to-agent payments in action: one agent produces intelligence, another agent consumes it, and settlement happens on-chain in seconds. The smart contract triage API becomes a self-sustaining service that scales with demand—no human intervention required.

FAQ

What is the x402 protocol?

The x402 protocol repurposes the HTTP 402 status code to enable machine-to-machine payments. When an agent requests a paid resource, the server responds with a 402 containing payment details (amount, wallet, network). The agent then submits an on-chain payment and retries the request with proof of payment.

How does Coinbase Bazaar enable agent commerce?

Coinbase CDP Bazaar provides a machine-indexed catalog of service SKUs that autonomous agents can discover and purchase. Sellers list their endpoints and pricing; buyer agents query the catalog, select matching services, and transact via x402—all without human involvement.

Why Base USDC for agent payments?

Base USDC offers sub-cent transaction fees, 2-second block times, and Coinbase-grade security. For microtransactions between agents—where payments might be fractions of a cent—Base USDC is the most efficient settlement rail available.

Can I use x402 with XRPL and T54?

Yes. The x402 protocol is network-agnostic. While Base USDC is the primary rail for Coinbase Bazaar, the protocol supports settlement on XRPL using the T54 token standard, enabling cross-chain agent commerce.
