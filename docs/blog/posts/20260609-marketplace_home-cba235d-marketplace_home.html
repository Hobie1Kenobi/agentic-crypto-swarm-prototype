How HTTP 402 Unlocks Agent Payments

The HTTP 402 status code has sat reserved in the spec for decades. The x402 protocol finally gives it a job: when an agent calls a paid endpoint, the server responds with 402 Payment Required and a payment header containing a settlement address and amount. The agent's client middleware catches the 402, initiates a micropayment on the appropriate rail, replays the request with a payment receipt, and receives the full response.

This flow is stateless, composable, and entirely programmatic. No API keys to rotate. No billing portals to configure. No monthly invoices to reconcile. Every request is self-contained: the agent pays, the seller delivers, the ledger confirms.

For builders, this means you can wrap any existing API in x402 payment middleware and start earning from autonomous buyers in minutes—not quarters. The x402 marketplace handles discovery, so agents find your tool the same way they find any other MCP-compatible resource.

Connect Agents via MCP + x402 in Under 5 Minutes

The Model Context Protocol (MCP) is becoming the standard wiring for agent tool use. By combining MCP with x402, you get paid tools that agents can discover and call natively—no custom billing integration required.

Here's the minimal connection path:

  • Step 1 — Register your tool: List your API on the marketplace with an OpenAPI spec, pricing in USDC or T54, and an MCP tool descriptor. The marketplace generates your x402 payment middleware automatically.
  • Step 2 — Point your agent: Add the marketplace's MCP server URL to your agent's configuration. Your agent now sees every paid tool as a callable function.
  • Step 3 — Fund the wallet: Deposit Base USDC or XRPL T54 into your agent's wallet. The agent handles the rest—catching 402s, paying, and retrying transparently.

That's it. From zero to a connected, transacting agent in under five minutes. The connect agents page has full integration guides for popular agent frameworks, and the API reference covers every endpoint.

Multi-Rail Settlement: Base USDC and XRPL T54

Agent commerce doesn't run on one chain. Different workloads have different requirements—cost, speed, finality, ecosystem. The Agentic Swarm Marketplace supports multi-rail settlement so agents can pay on the rail that makes sense for the transaction.

Base USDC

For high-throughput, low-latency workloads, Base USDC x402 payments settle in seconds on the Base L2. Smart contract triage APIs—where an agent needs to analyze hundreds of contracts per minute—benefit from Base's low gas costs and fast finality. Sellers receive USDC directly to their on-chain address, no intermediary.

XRPL T54

For cross-border, high-value, or compliance-sensitive transactions, XRPL T54 x402 payments leverage the XRP Ledger's native speed and negligible fees. T54 is designed for settlement scenarios where provenance and finality matter—think agent-to-agent payments between enterprise systems that require auditable, immutable transaction records.

Both rails are first-class citizens in the marketplace. Sellers set prices in either denomination; buyers pay on whichever rail they've funded. The x402 middleware handles routing automatically.

Smart Contract Triage for Autonomous Buyers

One of the most compelling early use cases on the marketplace is smart contract triage. When an agent encounters a new contract—whether it's auditing a DeFi protocol, screening a counterparty, or evaluating a deployment—it needs fast, reliable analysis.

Here's how it works in practice:

  • It calls a smart contract triage API listed on the marketplace via MCP.
  • The API responds with 402 and a payment request for 0.001 USDC.
  • The agent's x402 middleware pays on Base, replays the request, and receives a structured risk assessment.
  • The agent logs the result and continues its workflow—no human in the loop.

This pattern generalizes. Data enrichment, sentiment analysis, image generation, code review—any capability an agent might need can be packaged as a paid MCP tool and listed on the marketplace. Sellers earn per request. Buyers get exactly what they need, when they need it, with cryptographically verifiable payment.

FAQ

What is x402 and how does it differ from traditional API billing?

x402 uses the HTTP 402 status code to enable per-request, machine-to-machine payments. Instead of API keys and monthly invoices, agents pay automatically via on-chain settlement each time they call a paid endpoint. No accounts, no invoicing, no reconciliation—just request, pay, receive.

Which blockchains support x402 payments?

The Agentic Swarm Marketplace currently supports two settlement rails: Base USDC for high-throughput, low-cost transactions, and XRPL T54 for cross-border and compliance-sensitive payments. Both rails are accessible through the same x402 middleware, and agents can pay on whichever rail they've funded.

How do I list my API as a paid MCP tool?

Register on the marketplace, upload your OpenAPI spec and MCP tool descriptor, set your per-request price in USDC or T54, and the platform generates x402 payment middleware for your endpoint. Agents discover your tool through the marketplace's MCP server and pay per call automatically. Full instructions are on the connect agents page.

Is agent commerce the same as algorithmic trading?

No. Agent commerce is machine-paid consumption—agents purchasing API calls, data, and compute on a per-request basis. There is no trading, no speculation, and no financial advice. Every payment corresponds to a delivered service, settled on-chain for transparency.
