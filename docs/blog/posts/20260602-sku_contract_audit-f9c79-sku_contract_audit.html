Why Agent Commerce Needs Real-Time Contract Triage

The HTTP 402 protocol unlocked machine-to-machine payments, but payments without verification are just expensive mistakes. Consider the attack surface an autonomous buyer faces:

  • Rug vectors: Mint functions, hidden owner privileges, self-destruct paths that don't appear in the ABI.
  • Delegation traps: EIP-7702 delegates that can redirect execution to a malicious implementation after the transaction is confirmed.
  • Concentration risk: Token distributions where a single holder controls 80% of supply, ready to dump on liquidity pools.
  • Deployer reputation: Contracts deployed by addresses previously associated with exploits, even if the code itself looks clean.

A human auditor might catch these over a week of review. An agent operating in the Base USDC x402 economy needs the answer before it signs the transaction. The smart contract triage API delivers that answer as a first-class product—no PDFs, no Slack threads, just structured data.

Inside the Audit Stack: Slither + Echidna + On-Chain Intel

Static Analysis with Slither

Slither remains the fastest way to surface common vulnerability classes in Solidity code. Our pipeline runs Slither with a curated detector set optimized for DeFi and agent-commerce patterns—reentrancy, access control, unchecked returns, and state manipulation vectors. Results are normalized into the audit_report_v2 schema with severity classifications, source locations, and remediation hints that downstream agents can parse programmatically.

Property-Based Fuzzing with Echidna

Static analysis finds what's obviously wrong. Echidna finds what's subtly broken. We generate invariant assertions based on common DeFi safety properties—total supply consistency, withdrawal balance bounds, access control preservation—and let Echidna fuzz the contract with transaction sequences that break them. Any counterexample becomes a high-severity finding in the report, complete with the minimal reproduction sequence.

On-Chain Intelligence

Code review alone is insufficient when the threat model includes social engineering and deployment-time manipulation. The on-chain layer adds three critical dimensions:

  • Deployer profiling: We trace the deployer address across chains, flagging prior contracts, exploit associations, and funding sources.
  • EIP-7702 delegate detection: We identify whether any of the audited addresses have set delegation designators and, if so, resolve the delegate implementation and assess its risk profile.
  • Holder concentration analysis: We compute Gini coefficients and top-N holder percentages for token contracts, surfacing concentration risks that could trigger liquidity events.

All three layers run in parallel across up to three supplied addresses, and the results merge into a single coherent report.

Structured Output Built for Agent Pipelines

The defining feature of the audit_report_v2 format is that it was designed for machines, not humans. Every finding includes:

  • severity: numeric 0–4 scale (info, low, medium, high, critical) for deterministic gating logic.
  • category: machine-readable classification (reentrancy, access_control, delegation_risk, concentration, etc.).
  • affected_address: the specific contract from the input set.
  • evidence: source snippets, transaction hashes, or holder distributions—whatever is relevant, in a consistent schema.
  • remediation: actionable steps an agent or developer can take, formatted as structured data.

This means your agent can implement rules like: "If severity ≥ 3 in any category, abort the transaction" or "If holder concentration > 60% and deployer has no prior verified contracts, require manual review." The report becomes a composable building block in your agent commerce API stack.
