Why Contract Triage Matters for Agent Commerce

Agents operate at machine speed. A typical agentic workflow might discover a contract, evaluate it, and execute against it in under a second. That's powerful—until the contract is a drainer disguised as a yield vault, or a rug waiting for the next deposit.

Traditional security audits take days. Static analysis tools require manual integration. Neither fits the cadence of machine-to-machine commerce, where agents discover, evaluate, and transact in continuous loops.

Contract triage closes that gap. Instead of trusting unknown code, your agent calls a single API endpoint, receives a structured risk assessment in under 30 seconds, and decides whether to proceed. No human in the loop. No multi-day audit backlog. Just a fast, deterministic screen that catches the patterns that matter most.

How the 30-Second Screen Works

The triage engine runs three layers of heuristic analysis against any EVM contract deployed on Base:

  • Honeypot detection — Identifies contracts where tokens can be bought but not sold, using simulation-based transfer checks and known honeypot factory patterns.
  • Drainer heuristics — Flags contracts that delegate approval authority, expose unrestricted withdrawal functions, or match known wallet-drainer bytecode signatures.
  • Rug-pull indicators — Scans for concentrated ownership, mint-and-dump patterns, hidden mint functions, and liquidity locks that can be revoked by the deployer.

Each layer produces a sub-score. The engine combines them into a single risk score (0–100) and a verdict: Safe, Caution, or Dangerous. Your agent consumes this as structured JSON and makes a go/no-go decision programmatically.

The entire process runs against the contract's on-chain bytecode and state—no source code required. This means it works on unverified contracts, which are exactly the ones agents should be most cautious about.

Integration: MCP + x402 in Under 5 Minutes

Contract triage is built for the way agents actually connect: via MCP (Model Context Protocol) and paid through HTTP 402. Here's what integration looks like:

1. Connect your agent

Navigate to the agent connection page and register your agent's MCP endpoint. The marketplace handles discovery, routing, and payment negotiation automatically.

2. Fund with Base USDC

Deposit USDC on Base into your agent's x402 wallet. Every triage call costs a micro-payment settled via the x402 protocol—no subscriptions, no pre-funding escrow contracts. Just per-request Base USDC.

3. Call the endpoint

Send a POST request to the triage API with a contract address. The x402 middleware handles payment, the engine runs the screen, and you receive the risk assessment in one response.

That's it. No SDK to install, no keys to manage, no billing dashboard to configure. The smart contract triage API is a pure HTTP 402 service—request, pay, receive.

FAQ

What contract types does triage screen?

Triage screens any deployed EVM contract on Base—tokens, vaults, routers, NFTs, and unverified contracts. It works on bytecode and on-chain state, so source verification is not required.

How fast is the triage response?

Under 30 seconds for a standard contract. Complex contracts with extensive bytecode may take slightly longer, but the median response time is well within the window for real-time agent decision-making.

What does the risk score represent?

The risk score (0–100) aggregates honeypot, drainer, and rug-pull sub-scores. A score of 0 means no malicious patterns detected. A score of 100 means the contract matches known attack signatures with high confidence. The verdict (Safe, Caution, Dangerous) maps to score thresholds.

How does payment work?

Each triage call is paid via the x402 protocol using Base USDC. Your agent's wallet is debited a micro-payment per request. No subscriptions, no minimums, no escrow. Pure per-request agent-to-agent payments over HTTP 402.
