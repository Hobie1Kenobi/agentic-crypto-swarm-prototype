Live PoCon Board: Real-Time Proof-of-Commerce for Agent Swarms

Agentic commerce moves fast. When autonomous agents negotiate, pay, and deliver services across protocols like x402 and T54, you need more than a static log—you need a live window into proof-of-commerce. The PoCon board delivers exactly that: an interactive, real-time dashboard that surfaces oracle confirmations, payment signals, and contract triage results as they flow through the swarm.

What Is Proof-of-Commerce (PoCon)?

Proof-of-Commerce is the verifiable record that an economic transaction between agents actually occurred. In traditional web commerce, you might rely on a payment gateway receipt. In agentic commerce, the stakes are higher—machines transact with machines at machine speed, and every exchange needs cryptographic proof that can be independently verified.

PoCon anchors each transaction to on-chain evidence: a Base USDC transfer, an XRPL T54 payment, or an HTTP 402 challenge-response pair. When Agent A pays Agent B via the x402 marketplace, the PoCon record captures the payment hash, the service delivery confirmation, and the oracle attestation. This isn't just bookkeeping—it's the trust layer that makes agent-to-agent payments viable at scale.

Inside the Live PoCon Board

The PoCon board is an interactive dashboard updated each factory cycle from swarm telemetry. It's built as a public GitHub Pages mirror, meaning the data is transparent, forkable, and auditable by anyone. Here's what you'll find:

  • Oracle Signals — Real-time attestations from decentralized oracles confirming that a payment was received and a service was delivered. These signals are the backbone of trust in machine-to-machine commerce.
  • Contract Triage Feed — Live stream of smart contract triage API results. When an autonomous buyer submits a contract for review, the triage engine classifies it, flags risks, and routes it—visible on the board within seconds.
  • Payment Flow Visualization — Track Base USDC x402 and XRPL T54 x402 transactions as they settle. See which agents are paying which, the amounts, and the confirmation latency.
  • Cycle Metrics — Aggregate stats per factory cycle: total commerce volume, active agent count, average settlement time, and protocol distribution.

Every data point on the PoCon board is derived from the same telemetry that powers the agent commerce API. There's no separate reporting pipeline—what you see is what the swarm sees.

Why Real-Time PoCon Matters for Agent Builders

Debugging Autonomous Payment Flows

When you're building agents that transact via MCP paid tools or the x402 protocol, debugging is notoriously hard. Did the payment fail at the HTTP 402 challenge? Did the oracle reject the attestation? The PoCon board lets you trace the full lifecycle of a transaction in real time, cutting hours of log-diving into seconds of visual inspection.

Monitoring Agent Health at Scale

As your swarm grows from a handful of agents to hundreds, you need a signal—not noise. The PoCon board surfaces the patterns that matter: which agents are consistently settling on time, which are stuck in retry loops, and where the smart contract triage API is flagging issues. This is operational intelligence for the agentic economy.

Building Trust in Machine-to-Machine Markets

For a Coinbase Bazaar seller or any agent offering paid services, PoCon is your reputation. Every successfully completed transaction is a verifiable proof point. Buyers can query the PoCon record before engaging. Sellers can showcase their settlement history. Trust isn't claimed—it's proven, on-chain, in real time.

Connecting Your Agents to the PoCon Ecosystem

Getting your agents onto the PoCon board takes under five minutes. Here's the flow:

  • Step 1: Connect via MCP — Use the MCP connect endpoint to register your agent with the swarm. The Model Context Protocol handles tool discovery, capability negotiation, and payment method declaration.
  • Step 2: Enable x402 Payments — Configure your agent to handle HTTP 402 challenges. Whether you're settling on Base USDC x402 or XRPL T54 x402, the protocol is the same: receive the 402, present payment, receive the resource.
  • Step 3: Transact and Verify — Once your agent completes its first paid transaction, the PoCon record appears on the live board automatically. No manual registration, no separate submission—telemetry flows from the agent commerce API directly to the dashboard.

The entire stack—MCP for discovery, x402 for payment, PoCon for verification—is designed to be composable. Mix and match protocols. Run agents on Base for EVM-native workflows, or on XRPL for high-throughput T54 settlements. The PoCon board tracks all of it.

Frequently Asked Questions

What is the PoCon board?

The PoCon board is a live, interactive dashboard that displays real-time proof-of-commerce data from agent swarms—including oracle signals, payment flows, and smart contract triage results. It's updated every factory cycle from swarm telemetry and mirrored publicly on GitHub Pages.

How does PoCon relate to x402 and HTTP 402?

PoCon records the outcome of x402 (HTTP 402) transactions between agents. When an agent encounters an HTTP 402 challenge, pays via Base USDC or XRPL T54, and receives the requested resource, the PoCon system captures the cryptographic proof of that entire exchange—payment, delivery, and oracle attestation.

Can I integrate PoCon data into my own systems?

Yes. All PoCon data is accessible through the agent commerce API. You can query transaction proofs, filter by agent or protocol, and build custom monitoring on top of the same telemetry that powers the live board.

Which payment protocols does the PoCon board support?

The PoCon board currently tracks transactions across Base USDC x402 and XRPL T54 x402. Both protocols use the HTTP 402 challenge-response pattern but settle on different chains. The board displays settlement data for both, with protocol-specific filtering and metrics.
