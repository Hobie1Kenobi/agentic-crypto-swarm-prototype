The Stack: Foundry, LangGraph, x402, and MCP

Every swarm needs a foundation. Here's what the repo ships with:

  • Foundry contracts — Solidity smart contracts compiled and deployed via Foundry, handling escrow, settlement, and verification for agent-to-agent payments on Celo and Base.
  • LangGraph agents — Stateful, graph-orchestrated agents that reason over multi-step workflows: triage a contract, decide whether to pay, execute the transaction, and report results.
  • x402 sellers — HTTP 402 payment-gated endpoints that turn any API into a paid service. Agents present payment headers; sellers verify on-chain and respond. No API keys, no billing portals—just cryptographic proof of payment.
  • MCP server — A Model Context Protocol server that exposes paid tools to connected agents, enabling discoverability and structured invocation across the swarm.

Each layer is modular. Swap the LLM, change the chain, add a new tool—the architecture is designed to be forked, extended, and improved by the community.

Connect Agents via MCP + x402 in Under 5 Minutes

The fastest path from zero to transacting looks like this:

  1. Clone the repo — git clone the prototype and install dependencies.
  2. Start the MCP server — Run the server locally; it registers your paid tools and makes them discoverable to any connected agent.
  3. Spin up a LangGraph agent — Point it at the MCP server. The agent now sees available tools, their costs, and their input schemas.
  4. Enable x402 payments — Configure your agent's wallet (Celo or Base USDC). When the agent calls a paid tool, it attaches an x402 payment header automatically.
  5. Execute — The seller verifies payment on-chain, fulfills the request, and returns the result. End-to-end latency: seconds, not minutes.

This is the agent commerce API pattern in its purest form: agents discover capabilities through MCP, pay through x402, and settle on-chain. No human in the loop, no credit card forms, no invoicing. Machine-to-machine commerce at protocol speed.

Base USDC Contract Triage for Autonomous Buyers

One of the most compelling use cases in the repo is smart contract triage—an autonomous agent that evaluates on-chain contracts and decides whether to transact. Here's how it works on Base:

  • Ingest — The agent pulls contract bytecode and ABI from Base blockheaders.
  • Analyze — LangGraph orchestrates a multi-step reasoning flow: check proxy patterns, scan for known vulnerabilities, estimate gas efficiency.
  • Decide — Based on the analysis, the agent either proceeds with a Base USDC x402 payment or flags the contract for human review.
  • Transact — If the contract passes triage, the agent sends USDC on Base through an x402-gated endpoint and receives the service output.

This pattern—autonomous evaluation leading to autonomous payment—is the backbone of agentic commerce. The smart contract triage API in the repo is ready to deploy, and it's a perfect starting point for builders who want to create Coinbase Bazaar seller endpoints or integrate with the broader x402 marketplace.

Multi-Chain Payments: Celo-First, Base + XRPL Inbound

The prototype is Celo-first for development and testing—low gas fees, fast finality, and a developer-friendly EVM environment. But agent commerce is inherently multi-chain, and the architecture reflects that:

  • Celo — Primary chain for contract deployment, testing, and local agent wallets.
  • Base — Production-grade USDC settlement for high-throughput agent-to-agent payments. Base's low fees and Coinbase institutional rails make it ideal for the Coinbase Bazaar seller pattern.
  • XRPL — Inbound machine payments using the T54 standard. XRPL's speed and cost profile make it a strong fit for high-frequency microtransactions between agents.

The XRPL T54 x402 integration means agents can receive payments on XRPL while serving responses from any chain. Cross-chain settlement is handled by the Foundry contracts, which verify payment proofs regardless of origin chain.

Builders who want to add new chains—Arbitrum, Polygon, Solana—will find the contract architecture straightforward to extend. Contributions and bounties are welcome.

FAQ

What is x402 and how does it enable agent commerce?

x402 is the HTTP 402 Payment Required status code repurposed as a protocol for machine-to-machine payments. Instead of API keys or OAuth tokens, agents present cryptographic proof of on-chain payment in HTTP headers. Sellers verify the payment and respond. This eliminates billing infrastructure and enables fully autonomous transactions.

How does MCP connect to paid agent tools?

The Model Context Protocol (MCP) provides a standardized way for agents to discover and invoke tools. In this swarm, the MCP server registers paid tools with their schemas and costs. Connected agents query the server, see available tools, and call them—attaching x402 payment headers when required. The result: MCP paid tools that any compatible agent can access instantly.

Why Celo-first for development?

Celo offers sub-second block times, negligible gas fees, and a mobile-native EVM environment—ideal for rapid prototyping and testing. The prototype's contracts deploy on Celo by default, with Base and XRPL configurations for production settlement and inbound payments.

Can I contribute and earn bounties?
