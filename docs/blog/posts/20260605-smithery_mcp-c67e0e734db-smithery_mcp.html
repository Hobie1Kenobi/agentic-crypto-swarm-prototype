Smithery MCP Server: Connect Agents to Paid Tools via x402

Building autonomous agents that actually do things—triage smart contracts, analyze on-chain data, execute bounties—requires more than a prompt and a prayer. It requires tools. And until now, giving a remote agent access to a paid tool meant cloning a repo, configuring environment variables, managing API keys, and praying the dependency tree didn't implode. That friction kills agent-to-agent commerce before it starts.

Today we're publishing the agentic-swarm-marketplace MCP server on Smithery. Remote agents can now discover and invoke paid tools over the Model Context Protocol without cloning anything, without managing credentials manually, and without leaving their runtime. Paired with the x402 payment protocol and the T54 settlement stack, this turns every agent into a buyer in a machine-to-machine economy.

What the Smithery MCP Listing Unlocks

Smithery is the emerging registry for MCP servers—the package manager for agent tools. Listing there means any agent runtime that speaks MCP can find our tools, inspect their schemas, and call them in a single session. Here's what changes:

  • No repo cloning. Agents connect to the remote MCP endpoint directly. The server advertises available tools, their input schemas, and their per-call cost. The agent decides whether to pay and invoke.
  • Composable discovery. An agent building a smart contract triage pipeline can mix tools from our server with tools from any other Smithery-listed MCP server—on-chain data lookups, XRPL settlement, Base USDC transfers—all in one orchestration pass.
  • Instant monetization for tool builders. If you've wrapped a capability as an MCP tool and listed it through the x402 marketplace, every connected agent is a potential customer. No storefront, no billing integration—just HTTP 402 negotiation.

This is the infrastructure layer agentic commerce has been missing: a discovery protocol (MCP), a payment protocol (x402/HTTP 402), and a settlement rail (T54 + Base USDC) that work together without human intervention.

How It Works: MCP + x402 + T54 in Under 5 Minutes

The integration path is deliberately minimal. Here's the full flow from zero to a paid tool invocation:

1. Connect Your Agent

Point your agent runtime at the MCP endpoint. Whether you're using Claude, an OpenAI agent, or a custom orchestrator, the connection is a single URL:

Endpoint: https://api.agentic-swarm-marketplace.com/mcp

The server responds to initialize and tools/list just like any standard MCP server. No SDK install, no build step.

2. Discover Paid Tools

Call tools/list. The response includes every available tool—smart contract triage, on-chain analytics, bounty fulfillment—along with each tool's input schema and its per-invocation price denominated in USDC on Base.

3. Invoke with x402 Payment

When your agent calls a tool, the server returns HTTP 402 Payment Required with a payment header pointing to the x402 facilitator. The agent's x402 handler signs a Base USDC transfer, the T54 stack verifies and settles, and the tool executes. Total round-trip: typically under two seconds.

That's it. Three steps. No API keys to rotate, no billing dashboards to configure, no invoice to reconcile at month-end. The agent commerce API handles the entire lifecycle from discovery through settlement.

Base USDC Contract Triage for Autonomous Buyers

One of the most requested capabilities on the marketplace is smart contract triage—the ability for an agent to ingest a contract address, pull its bytecode and storage layout, classify its type (DeFi, NFT, governance, proxy), flag known vulnerability patterns, and return a structured risk assessment. All without a human in the loop.

Our MCP server exposes this as a first-class tool. Here's the machine-to-machine flow:

  • Agent receives a contract address (from a user prompt, a webhook, or another agent's output).
  • Agent calls the triage tool via MCP, paying the per-call fee via x402 in Base USDC.
  • T54 settlement confirms the payment on the XRPL side, and the tool returns a structured JSON report: contract type, risk flags, dependency graph, and recommended next actions.

For agents operating on Coinbase CDP Bazaar or any Base-native workflow, this is the missing piece: an on-demand, pay-per-use intelligence layer that doesn't require pre-negotiated contracts or volume commitments. An autonomous buyer can triage 100 contracts in an hour and pay exactly for what it uses.

The pricing is transparent because it has to be—agents can't negotiate. Every tool on the server publishes its cost upfront. The x402 protocol ensures the agent pays exactly that amount, and the T54 stack ensures the seller receives it. No middlemen, no float, no disputes.

Frequently Asked Questions

  • What is the agentic-swarm-marketplace MCP server?
    It's a remote MCP server listed on Smithery that exposes paid tools—smart contract triage, on-chain analytics, bounty fulfillment—to autonomous agents. Agents connect via the standard MCP protocol and pay per invocation using the x402 (HTTP 402) payment protocol.
  • How do agents pay for tools?
    When an agent invokes a paid tool, the server returns HTTP 402 with an x402 payment header. The agent's x402 handler signs a Base USDC transfer, the T54 settlement stack verifies it, and the tool executes. No API keys or manual billing required.
  • Do I need to clone a repository to use these tools?
    No. The MCP server is fully remote. Agents connect to the endpoint, discover tools via tools/list, and invoke them directly. No cloning, no local dependencies, no build steps.
  • What is T54 and how does it relate to x402?
    T54 is the settlement stack that verifies and finalizes x402 payments. While x402 handles the HTTP 402 negotiation and payment signaling, T54 ensures the Base USDC (and XRPL) transfers are confirmed and the seller is paid before the tool result is delivered.
