The Problem: Agents Can't Read Intent

Modern agent frameworks are remarkably good at composing transactions, routing through DEXs, and managing portfolios. What they're terrible at is intent analysis. An ERC-20 token contract can look perfectly standard on the surface while hiding a tax mechanism that makes selling impossible. A liquidity pool can appear deep while the contract owner retains a mint function that inflates supply on demand.

These aren't edge cases. On Base alone, the volume of newly deployed contracts means agents encounter unvetted code dozens of times per session. Manual review doesn't scale. Even automated static analysis tools designed for developers take minutes to run and require interpretation. Agents need something faster and more decisive.

How Contract Triage Works

The smart contract triage API operates as a pre-flight gate: a lightweight, high-speed screen that catches the most common and most damaging malicious patterns before your agent interacts with a contract.

Pattern Detection Engine

The triage engine evaluates contracts across three primary threat categories:

  • Honeypot heuristics — Detects buy-only tax logic, transfer restrictions that prevent selling, and balance-manipulation functions that lock user funds. These patterns are the most common trap for autonomous buyers.
  • Drainer detection — Identifies approval-exploitation patterns, unlimited spend allowances, and delegate-call vectors that enable wallet-draining attacks. Critical for any agent that grants token approvals.
  • Rug-pull indicators — Flags owner-controlled mint functions, renounceable-but-not-yet-renounced privileges, liquidity-lock anomalies, and contract patterns matching known rug deployments.

Each category contributes to a composite risk score from 0 (clean) to 100 (critical threat), accompanied by a human-readable verdict: Safe, Caution, or Avoid.

The 30-Second Guarantee

Speed matters because agent loops are tight. A 30-second triage window means your agent can call the screen, receive the verdict, and make a go/no-go decision within a single execution cycle—no timeout, no stale data. The API achieves this through pre-indexed pattern libraries and on-chain state snapshots rather than full bytecode symbolic execution, trading exhaustive depth for actionable speed.

Building Safer Agent Commerce

Contract triage isn't a silver bullet—sophisticated zero-day contract exploits can evade pattern matching. But it eliminates the obvious threats, which account for the vast majority of losses in on-chain agent operations. Think of it as the seatbelt: it won't prevent every accident, but it dramatically reduces the damage from the most common ones.

For agent builders running market-making strategies, portfolio rebalancing, or autonomous airdrop farming on Base, contract triage is the difference between a profitable loop and a catastrophic one. The agent commerce API ecosystem depends on trust—agents need to trust the contracts they interact with, and operators need to trust that their agents won't self-destruct on the first malicious token they encounter.

This is especially critical as machine-to-machine payments proliferate. When Agent A pays Agent B to execute a swap, the payment chain is only as safe as the weakest contract in the path. Triage at the entry point hardens the entire chain.

What chains does contract triage support?

Contract triage currently screens EVM contracts deployed on Base. Support for additional chains is on the roadmap, but Base's high throughput and low fees make it the priority for agent commerce workloads.

How accurate is the risk score?

The triage engine catches over 95% of known honeypot, drainer, and rug-pull patterns based on historical deployment data. It is a heuristic screen, not a formal verification tool—novel or highly obfuscated exploits may not be flagged. Always pair triage with additional due diligence for high-value transactions.

How does x402 payment work for the triage API?

Each triage request uses the HTTP 402 protocol. Your agent includes a Base USDC micropayment with the API request. No API keys or subscriptions are required—payment and request are atomic. If the payment is valid, the triage result is returned immediately.

Can I use contract triage with non-agent workflows?

Yes. Any system that can make HTTP requests and handle x402 payments can call the triage API. This includes backend services, CI pipelines for contract deployment checks, and manual developer tools—not just autonomous agents.
