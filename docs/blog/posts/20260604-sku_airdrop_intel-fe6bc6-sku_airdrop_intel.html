Airdrop Intelligence: Screen Scams Before They Screen You

Airdrop intelligence gives agents a Farm Score from 0 to 100, filtering scams before any wallet connects. Pure analysis, zero on-chain execution.

Airdrops are the original double-edged sword of crypto. On one side: legitimate protocol distributions that reward early adopters and bootstrap network effects. On the other: sophisticated phishing operations, Sybil farms, and honeypot contracts dressed up in Discord-ready branding. For autonomous agents operating in the x402 marketplace, telling the difference isn't optional—it's existential.

That's why we built Airdrop Intelligence: a constitution-first screening layer that scores any airdrop from 0 (probable scam) to 100 (high-confidence legitimate) before a single signature is requested. No on-chain execution. No wallet connection. Just clean, actionable intelligence delivered via the agent commerce API your swarm already trusts.

Why Airdrop Screening Needs a Constitution

Most airdrop checkers run a grab-bag of heuristics—contract age, liquidity depth, social follower counts—and spit out a binary "safe" or "unsafe." That's not enough for machine-to-machine decisions. Agents need a constitution: a transparent, versioned set of rules that determines exactly how a score is computed, so you can audit, override, or extend it.

Our constitution encodes hard rejection criteria first:

  • Unverified contract source — if the bytecode doesn't match a verified source on-chain, the airdrop is flagged immediately.
  • Known phishing patterns — signature-draining approve() calls, infinite allowance requests, and transferFrom hooks that drain rather than deposit.
  • Sybil indicators — duplicate claim patterns across wallets that map to the same cluster.
  • Impersonation signals — domain squatting, copied branding, and social graph anomalies.

Only after an airdrop passes these hard blocks does the scoring layer kick in, weighing factors like team reputation, token distribution fairness, and historical claim behavior. The result is a single Farm Score from 0 to 100 that your agent can threshold against: "only interact with airdrops scoring 70+" or "flag anything below 40 for human review."

Intelligence-Only Architecture: No On-Chain Execution

Here's the part engineers appreciate: Airdrop Intelligence never touches a wallet or submits a transaction. It's an intelligence-only service. You send a contract address, a claim URL, or a token identifier; you receive a structured JSON response with the Farm Score, constitution rule hits, and an optional triage embed. Zero on-chain side effects.

This design choice matters for three reasons:

  • Security surface area — no private keys, no transaction signing, no gas. The attack surface shrinks to an API endpoint.
  • Composability — your agent can layer this intelligence into any workflow: a Discord bot that warns users, a portfolio manager that skips junk tokens, or a MCP paid tool that charges per screening via the x402 protocol.
  • Speed — without waiting for block confirmations or RPC latency, screening returns in milliseconds, not seconds.

The API speaks HTTP 402 natively. When your agent calls the endpoint, the x402 handshake handles micropayment in Base USDC automatically—no API keys, no billing portals, no invoice reconciliation. Machine-to-machine commerce, exactly as it should be.

Optional Contract Triage on Base

For teams that want to go deeper than a Farm Score, we offer an optional smart contract triage API embed. This layer performs static analysis on the contract bytecode: it traces function call graphs, identifies dangerous patterns (reentrancy, unchecked external calls, hidden mint functions), and returns a structured triage report.

The triage embed runs on Base for a reason. Base's low gas costs and high throughput make it economical to spin up isolated analysis environments per request. Your agent pays a few cents in Base USDC via the x402 protocol and receives a full triage report—function signatures, risk annotations, and a confidence score on the contract's safety.

This is especially powerful for agent-to-agent payments. Imagine an agent marketplace where one agent specializes in contract triage and sells that capability to other agents on-demand. The triage agent screens the contract, the buying agent pays for the result, and both parties settle instantly through the x402 layer. No humans in the loop, no manual invoicing.

FAQ

What is a Farm Score?

A Farm Score is a 0–100 rating produced by our constitution-first screening engine. A score of 0 indicates a high probability of scam or Sybil farm; 100 indicates a high-confidence legitimate airdrop. Agents use this score to set thresholds for automated interaction or human escalation.

Does Airdrop Intelligence execute any on-chain transactions?

No. Airdrop Intelligence is an intelligence-only service. It never connects to a wallet, signs a transaction, or submits anything on-chain. It analyzes publicly available data and returns structured scoring and triage information.

How do agents pay for screening?

Agents pay per screening call using the HTTP 402 (x402) protocol. Payment settles in Base USDC automatically through the agent commerce API—no API keys or billing dashboards required. For XRPL-based agents, the T54 rail supports the same flow.

Can I use the contract triage embed on chains other than Base?

The triage analysis can evaluate contracts from any EVM-compatible chain, but the analysis environment and payment settlement currently run on Base for cost efficiency. You don't need your agent to operate on Base to use the service—only the payment rail runs there.
