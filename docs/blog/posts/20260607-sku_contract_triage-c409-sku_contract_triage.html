The Threat Landscape for Autonomous Agents

The rise of agentic commerce—machine-to-machine transactions powered by protocols like x402 and MCP—has created an entirely new attack surface. Agents operate at machine speed, executing swaps, granting approvals, and transferring Base USDC without human confirmation. Malicious contract developers know this. They've built an arsenal of patterns designed to exploit automated execution:

  • Honeypot contracts that simulate legitimate yield or token interactions but prevent withdrawals through hidden logic in transfer functions.
  • Drainer contracts that exploit approval patterns, sweeping ERC-20 tokens and NFTs once an agent signs a seemingly innocuous transaction.
  • Rug-pull tokens with mint functions, pause mechanisms, or hidden fee structures that activate only after sufficient liquidity is trapped.

For a human trader, these patterns are sometimes visible in the bytecode. For an autonomous agent executing at HTTP 402 speed, they're invisible—and that invisibility is expensive. A single drainer interaction can cost thousands in Base USDC before the agent's next decision cycle.

How the 30-Second Screen Works

Contract Triage runs a multi-layer heuristic analysis against any EVM contract address on Base, returning a structured verdict in under 30 seconds. The screen is designed for machine consumption—no dashboards to interpret, no subjective judgment calls. Just a risk score and a clear verdict your agent can act on programmatically.

Pattern Detection Engine

The triage engine checks against a continuously updated library of malicious patterns:

  • Honeypot heuristics: Transfer restrictions, balance manipulation, and fake-quote mechanisms that simulate tradable tokens while blocking exits.
  • Drainer signatures: Known drainer contract bytecodes, suspicious approval patterns, and function signatures associated with wallet-draining exploits.
  • Rug indicators: Hidden mint functions, owner-controlled pause mechanisms, concentrated liquidity traps, and fee structures that escalate post-purchase.
  • Proxy and upgrade risks: Contracts with delegatecall patterns, upgradeable proxies, or storage collision vulnerabilities that could change behavior after the initial screen.

Risk Score and Verdict

Every triaged contract receives a risk score from 0 to 100 and one of three verdicts:

  • SAFE — No malicious patterns detected. The contract passes all heuristic checks.
  • CAUTION — Suspicious patterns detected that don't meet the threshold for a malicious classification but warrant additional scrutiny.
  • MALICIOUS — Known malicious patterns confirmed. The contract should not be interacted with.

This structure maps directly to agent decision logic. An agent receiving a MALICIOUS verdict aborts the transaction. A CAUTION verdict triggers a secondary verification step or a human-in-the-loop escalation. SAFE allows the agent to proceed with confidence.

Pre-Flight for Agent Commerce Flows

Contract Triage is designed as a pre-flight check—the step your agent takes before it touches any unknown contract. Here's how it fits into common agent commerce patterns:

Token Swaps and DEX Interactions

Before an agent executes a swap on a Base DEX, it screens the target token contract. If the triage returns MALICIOUS, the agent skips the swap and logs the contract address. This prevents honeypot tokens from trapping agent funds.

Approval Management

Before granting an ERC-20 approval to any contract, the agent runs triage on the spender address. Drainer contracts are flagged before the approval transaction is ever signed.

Agent-to-Agent Payments

In agent-to-agent payment flows—where one agent pays another for data, compute, or services—Contract Triage screens the receiving contract. This is especially relevant for Coinbase Bazaar sellers and marketplace participants who need to verify that the contracts they're interacting with are legitimate before committing Base USDC.

Cross-Chain Considerations

The goal is a universal pre-flight layer for any contract an autonomous agent might touch.
