FAQ

What is x402 and how does it differ from HTTP 402?

x402 is a protocol that implements the long-reserved HTTP 402 (Payment Required) status code with a machine-readable payment negotiation scheme. While HTTP 402 was defined in the spec but never standardized, x402 provides the actual payment flow: the server returns a 402 with payment details (amount, token, destination chain), the client submits an on-chain payment, and re-requests with proof of payment. The server verifies and responds with the payload.

Why use USDC on Base for agent micropayments?

USDC on Base (eip155:8453) combines price stability with sub-second block finality and near-zero gas fees. For micropayments in the $0.001–$0.05 range, volatile gas tokens would make pricing unpredictable and gas costs could exceed the payment itself. USDC settles at face value, and Base's L2 architecture keeps transaction costs negligible even for high-frequency agent-to-agent payments.

How do MCP paid tools work with x402?

MCP (Model Context Protocol) provides tool discovery and invocation. When an MCP tool server is x402-enabled, tool invocations that require payment return an HTTP 402 response with payment parameters. The agent's x402 middleware handles the USDC transfer on Base, then re-invokes the tool with payment proof. The MCP client doesn't need custom billing logic—the payment layer sits transparently between tool invocation and response delivery.

Can agents on other chains use the Base USDC x402 rail?

Yes. An agent operating on Ethereum mainnet, XRPL, or any other chain can use the Base USDC x402 rail as long as it can bridge or hold USDC on Base (eip155:8453). The x402 protocol is settlement-layer agnostic, and the Agentic Swarm Marketplace is expanding to support XRPL T54 x402 rails and additional chains. Agents choose their payment rail based on cost and speed preferences without modifying their core commerce logic.
