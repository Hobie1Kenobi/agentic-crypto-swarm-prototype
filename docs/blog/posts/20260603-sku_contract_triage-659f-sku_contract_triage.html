The Problem: Agents Move Fast, Threats Move Faster

When an agent discovers a new liquidity pool, a yield vault, or an NFT marketplace contract on Base, it faces a trust decision. Traditional approaches—manual audit links, community reputation scores, or static allowlists—don't scale in an agentic economy where agents discover and interact with contracts in milliseconds.

The threat surface is real and growing:

  • Honeypot contracts that accept deposits but revert on withdrawal, trapping USDC and other tokens permanently.
  • Drainer contracts that exploit approval patterns to sweep a wallet's entire balance the moment an agent signs an innocuous-looking permit.
  • Rug-pull patterns where deployer-controlled functions can mint, burn, or pause tokens at will—often hidden behind proxy patterns or obscured ownership.

For human traders, a bad interaction means a loss. For autonomous agents operating at machine speed across the x402 marketplace, a bad interaction can compound into catastrophic exposure before any alert fires.

How Contract Triage Works

The contract triage endpoint performs a multi-layered heuristic analysis of any EVM contract on Base and returns results in under 30 seconds. Here's what happens in that window:

1. Bytecode Pattern Matching

The system decompiles and scans contract bytecode against a continuously updated library of known malicious signatures—honeypot withdrawal guards, hidden mint functions, self-destruct opcodes, and proxy manipulation patterns. This catches the majority of copy-paste scams that reuse known attack templates.

2. State-Dependent Heuristics

Some threats only emerge at runtime. The triage engine simulates key transaction paths—deposits, withdrawals, approvals, transfers—against the contract's current on-chain state. If a withdrawal reverts under simulated conditions while a deposit succeeds, the honeypot flag triggers immediately.

3. Ownership and Permission Analysis

Contracts with mutable state, upgradeable proxies, or concentrated ownership pose elevated risk. The triage engine maps the permission graph: who can call what, which functions are guarded by role-based access, and whether ownership is renounced or concentrated in a single address.

4. Risk Score and Verdict

All signals collapse into a single structured response:

  • Risk score (0–100): a calibrated probability that the contract exhibits malicious behavior.
  • Verdict: Safe, Caution, or Malicious—a clear, actionable signal your agent can branch on programmatically.
  • Flags: specific patterns detected (e.g., honeypot_withdrawal_guard, unlimited_approval_drain, owner_mint_capability).

This isn't a fuzzy confidence interval. It's a decision-ready output designed for machine consumption.

No credit cards. No rate-limit negotiations. No shared API keys leaking between agent instances. The HTTP 402 flow means every request carries its own payment, and every payment is settled on-chain in seconds.

When to Run Triage

The 30-second screen is designed as a pre-flight gate, not a post-mortem. Run it:

  • Before any first interaction with a contract your agent hasn't previously screened.
  • Before large-position entries where even a low-probability threat creates outsized exposure.
  • On recurring schedules for contracts your agent interacts with regularly—proxy upgrades and ownership changes can turn a previously safe contract malicious overnight.
  • When discovering new contracts through on-chain exploration, DEX routing, or agent-to-agent referrals in the x402 marketplace.

The cost of a triage call is measured in micro-USDC. The cost of skipping one is measured in the full value of your agent's wallet. The math is straightforward.

FAQ

What chains does contract triage support?

Currently, the triage engine screens EVM contracts deployed on Base. Support for additional EVM-compatible chains and XRPL T54 x402 integration is on the roadmap.

How accurate is the risk score?

The heuristic engine combines bytecode pattern matching, state-dependent simulation, and permission analysis. It catches the vast majority of known honeypot, drainer, and rug patterns. Novel or highly obfuscated attacks may not be flagged—triage is a risk screen, not a formal audit. Use it as a pre-flight gate, not a guarantee.

How much does a triage call cost?

Each call is settled via the x402 protocol on Base USDC. Pricing is micro-USDC per request—fractions of a cent. No subscriptions, no API keys, no minimums. Your agent pays per call and settles on-chain.

Can I use contract triage without MCP?

Yes. The endpoint is a standard x402-paid HTTP API. You can call it directly from any agent runtime that supports HTTP 402 payment flows. MCP integration is the fastest path for agents already in the ecosystem, but it's not required.
