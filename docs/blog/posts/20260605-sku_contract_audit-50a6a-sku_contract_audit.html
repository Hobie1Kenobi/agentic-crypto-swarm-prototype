Why Traditional Audits Break Down for Agent Pipelines

Manual audits serve a critical function for protocol launches and governance reviews. But they operate on human timescales—days to weeks—and produce narrative documents that require interpretation. When an autonomous buyer on Base USDC x402 rails evaluates a contract before committing capital, it needs deterministic, machine-parseable answers immediately.

The friction points are real:

  • Latency: PDF reports can't be ingested by agents in sub-second decision loops.
  • Ambiguity: Severity ratings buried in prose require NLP parsing, introducing hallucination risk.
  • Scope gaps: Most audits examine a single deployed bytecode, ignoring deployer behavior, delegate changes, and holder concentration—all critical signals for autonomous risk assessment.

The x402 marketplace demands audit infrastructure that speaks the same protocol language as the agents consuming it. That means structured JSON, deterministic scoring, and multi-dimensional on-chain context.

Inside the Audit: Slither + Echidna + On-Chain Intel

The Full Contract Audit endpoint orchestrates three complementary analysis layers into a single coherent report. Here's how each contributes to the final risk picture.

Slither: Static Analysis at Speed

Slither remains the fastest path to surface common vulnerability classes—reentrancy, uninitialized storage, access control failures, and dozens of other patterns. The audit endpoint runs Slither against your target contracts with optimized detector configurations, filtering noise and mapping findings to standardized severity buckets. For agents, this means immediate signal on whether a contract contains known antipatterns before committing deeper analysis resources.

Echidna: Fuzzing for Invariant Violations

Static analysis catches what's structurally wrong. Echidna catches what's behaviorally wrong. By fuzzing contract invariants under adversarial conditions, Echidna surfaces logic errors that Slither can't see—state inconsistencies, unexpected token flows, boundary condition failures. The audit endpoint configures Echidna campaigns with targeted invariant sets and runs them to convergence or timeout, surfacing any violations as structured findings with reproduction paths.

On-Chain Intelligence: Deployer Profiling, Delegates, and Concentration

This is where the audit transcends code review and becomes on-chain triage. Across up to three specified addresses, the endpoint profiles:

  • Deployer history: Prior deployments, associated contracts, known exploit links, and behavioral patterns across chains.
  • EIP-7702 delegate detection: Identifies whether the target contract has delegated execution authority via EIP-7702, what delegate contracts are involved, and whether those delegates introduce additional attack surface.
  • Holder concentration: Token distribution analysis revealing whale concentration, liquidity depth, and rug-pull risk indicators.

These on-chain signals are often the strongest predictors of real-world exploit risk—far more actionable than theoretical code vulnerabilities in isolation.

Structured Output: audit_report_v2 for Agent Pipelines

The entire audit output conforms to the audit_report_v2 JSON schema—designed from the ground up for machine consumption. No PDF parsing. No NLP extraction. No ambiguity.

A typical report includes:

  • findings[] — Array of structured vulnerability objects with severity, detector source (Slither/Echidna/on-chain), contract scope, and reproduction details.
  • deployer_profile — Address history, cross-chain behavior, risk flags.
  • delegate_analysis — EIP-7702 delegate mappings and associated risk assessments.
  • concentration_metrics — Holder distribution stats, top-holder percentages, liquidity ratios.
  • overall_risk_score — Composite deterministic score (0–100) aggregating all findings and on-chain signals.

This structure plugs directly into agent decision frameworks via MCP paid tools or custom pipeline integrations. An agent evaluating a Base USDC x402 transaction can query the endpoint, receive the JSON, and make a go/no-go decision in a single execution step—no human in the loop.

FAQ

What contracts and chains does the Full Contract Audit support?

The audit endpoint supports any EVM-compatible contract. On-chain intelligence layers (deployer profiling, holder concentration) are richest for Base, Ethereum mainnet, and other major EVM chains with indexed data. You can specify up to three contract addresses per audit request.

How does EIP-7702 delegate detection work?

The endpoint analyzes the target contract's code and storage to identify EIP-7702 delegation patterns—specifically, whether the contract has set a delegation designator pointing to an external implementation. It then profiles the delegate contract for known vulnerabilities, upgrade patterns, and behavioral risks, surfacing findings in the delegate_analysis section of the report.

What is the audit_report_v2 JSON schema?

audit_report_v2 is a structured JSON schema containing findings arrays, deployer profiles, delegate analysis, concentration metrics, and an overall deterministic risk score. It's designed for direct machine consumption—no parsing of human-readable documents required. The schema is versioned and backward-compatible.

How do I pay for audits as an autonomous agent?

Payments settle via the x402 (HTTP 402) protocol using Base USDC. Your agent makes a standard HTTP request, receives a 402 challenge with payment details, settles the payment on-chain, and retries the request with the payment proof. The entire flow is automated and requires no API keys or manual billing.
