For autonomous agents operating in agentic commerce, the stakes are even higher—an agent that claims a poisoned airdrop doesn't just lose funds; it loses credibility with every downstream transaction it touches.

That's why we built Airdrop Intelligence: a constitution-first screening layer that scores any airdrop from 0 (almost certainly a scam) to 100 (high confidence, clean) before a single transaction is signed. It's intelligence only—no on-chain execution, no wallet connection, no exposure surface. Just a clean signal your agent can act on.

Why Airdrop Screening Needs Its Own Layer

Airdrops are a unique attack vector. Unlike typical DeFi interactions where a user deliberately navigates to a protocol, airdrops arrive unsolicited. The attacker chooses the timing, the payload, and the social engineering wrapper. Traditional security tooling—transaction simulators, approval scanners—assumes you've already decided to interact. Airdrop intelligence flips that assumption: screen first, interact never unless the signal is clean.

For agent builders wiring into the x402 marketplace, this is a critical distinction. Your agent might be autonomous, but it shouldn't be gullible. A Farm Score of 15 on an airdrop means the agent should hard-stop—no claim, no contract interaction, no token transfer. A Farm Score of 85 means the airdrop has passed constitutional checks and is worth evaluating further.

How Farm Score Works: Constitution-First Scam Screening

Farm Score is the core output of the Airdrop Intelligence API. It runs on a constitution-first methodology: a set of hardcoded safety principles that are evaluated before any heuristic or statistical model. The constitution includes rules like:

  • No unsolicited approval requests — if the airdrop contract asks for token approvals at claim time, it fails.
  • No delegate call patterns — contracts using DELEGATECALL in claim paths are flagged immediately.
  • No self-destruct or mint-to-zero patterns — common rug-pull signatures are checked first.
  • Creator reputation weighting — known deployer addresses, ENS history, and on-chain behavior are factored in.

Only after the constitutional checks pass do secondary heuristics kick in: token distribution patterns, liquidity depth on DEXs, social signal correlation, and historical airdrop behavior from the same deployer. The result is a single integer—Farm Score 0–100—that your agent can threshold against in milliseconds.

Because this is intelligence only, there's no on-chain execution. Your agent never connects a wallet, never signs a transaction, and never exposes a public key. The API returns a JSON payload with the Farm Score, flagged risk factors, and an optional contract triage embed for deeper analysis.

Optional Contract Triage on Base

For airdrops that pass the initial Farm Score screen, you can optionally invoke a deeper smart contract triage API that disassembles the claim contract's bytecode, traces execution paths, and identifies hidden logic. This is where the real defense-in-depth lives.

Contract triage runs on Base and is paid via Base USDC x402. It's designed for autonomous buyers—agents that need to make go/no-go decisions without human intervention. The triage output includes:

  • Function signature map — every external-facing function in the contract.
  • State mutation graph — which functions modify storage and how.
  • Privilege escalation flags — any onlyOwner or role-based gates that could be weaponized.
  • Simulated claim path — a dry-run of the claim function with a mock wallet to detect hidden transfer or approval calls.

This is the same triage layer that Coinbase Bazaar sellers use to vet contracts before listing. Now it's available to any agent in the swarm.

FAQ

What is Farm Score?

Farm Score is a 0–100 integer that represents the scam likelihood of an airdrop. It's generated by constitution-first screening: hardcoded safety rules are evaluated first, then secondary heuristics. A score of 0 means almost certainly a scam; 100 means high confidence the airdrop is clean.

Does the Airdrop Intelligence API execute any on-chain transactions?

No. The API is intelligence only. It never connects to a wallet, signs a transaction, or interacts with a contract on-chain. It returns a screening signal that your agent uses to decide whether to proceed.

How do agents pay for airdrop screening?

Agents pay per request using the x402 protocol. Payments settle as Base USDC micropayments. For agents on XRPL, the T54 standard provides equivalent payment rails. No subscriptions or pre-funding required.

What's the difference between Farm Score screening and contract triage?

Farm Score screening is the fast, first-pass check—it evaluates the airdrop against constitutional rules and heuristics without deep contract analysis. Contract triage is an optional second step that disassembles the claim contract's bytecode, traces execution paths, and identifies hidden logic. Triage is paid separately via Base USDC x402.
