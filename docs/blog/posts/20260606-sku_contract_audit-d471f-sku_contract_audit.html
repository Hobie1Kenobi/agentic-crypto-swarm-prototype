Why Contract Triage Belongs in the Agent Stack

Autonomous agents executing trades, bridging assets, or settling bounties need deterministic, machine-readable safety signals before they sign transactions. A human-readable PDF audit is useless to a pipeline. What agents need is structured, versioned JSON they can parse, filter on, and chain into downstream decisions—approve, flag, or halt.

That's exactly what audit_report_v2 delivers. It's a normalized schema covering static analysis findings, fuzzing results, and on-chain intelligence, designed so your agent logic can consume it without a human in the loop.

What the Audit Covers

Static Analysis with Slither

Slither runs its full detector suite across your target contracts—reentrancy, uninitialized storage, access-control gaps, and dozens of other vulnerability classes. Results are mapped to severity levels and CWE references, then folded into the report JSON so your agent can filter on finding.severity >= "high" and act immediately.

Fuzzing with Echidna

Echidna exercises your contract's invariant properties with coverage-guided fuzzing. The audit endpoint ships pre-built Echidna corpora for common patterns (ERC-20, ERC-721, proxy upgradeability) and can accept custom invariant definitions. Failing invariants surface as structured findings, complete with minimized reproduction sequences your agent can replay or log.

On-Chain Intel Across Up to 3 Addresses

Static analysis and fuzzing tell you what the code could do. On-chain intel tells you what the deployer and holders have done. The audit endpoint profiles:

  • Deployer history — past contracts, rug-pull signals, team multisig linkage.
  • EIP-7702 delegate detection — identifies if an EOA has delegated authority to a contract code hash, a critical surface for post-EIP-7702 Ethereum.
  • Holder concentration — top-N holder percentages, whale clustering, and liquidity depth checks.

All three dimensions are merged into a single audit_report_v2 object, ready for programmatic consumption.

FAQ

How many contract addresses can I audit in a single request?

Up to 3 addresses per request. The on-chain intel module cross-references deployer and holder data across all submitted addresses, giving you a consolidated view rather than three isolated reports.

What is audit_report_v2?

It's a versioned JSON schema that normalizes findings from Slither, Echidna, and on-chain analysis into a single machine-readable object. Each finding includes severity, CWE reference, location, and—where applicable—a minimized reproduction trace. The schema is designed for deterministic parsing by agent pipelines.

Does the audit detect EIP-7702 delegation?

Yes. The on-chain intel module specifically checks for EIP-7702 delegate contracts on submitted EOAs, flagging delegated code hashes and their associated risk signals. This is critical for agents interacting with post-Pectra Ethereum accounts.

How does x402 payment work for this endpoint?

The endpoint uses the HTTP 402 protocol. Your agent includes a Base USDC micropayment in the request header. The x402 layer verifies settlement, then the audit runs and returns the report. No API keys or subscription management required—pure pay-per-use, machine-to-machine commerce.
