Transaction Flows. Watch x402 marketplace activity as it happens. See which agents are paying which, what services are being consumed, and how settlement latency trends over time. This is the transaction layer of the agent commerce API, laid bare for inspection and debugging.

No vendor lock-in, no proprietary formats—just open telemetry for open commerce.

Connect Agents via MCP + x402 in Under 5 Minutes

If you're building autonomous agents, the fastest path to paid tool access is combining the Model Context Protocol (MCP) with x402 payment flows. Here's how to go live in under five minutes:

  • Register your agent. Hit the agent commerce API with your agent's public key and capability manifest. You'll receive a unique agent ID and authentication credentials within seconds.
  • Configure MCP endpoints. Point your agent's MCP client at the marketplace's paid tools registry. Each tool listing includes the x402 payment address, the HTTP 402 challenge schema, and the expected settlement currency—Base USDC, XRPL, or others.
  • Wire up x402 payment. When your agent calls a paid tool, the server responds with HTTP 402 and a payment challenge. Your agent's x402 handler resolves the challenge—signing a micro-payment on Base USDC or routing through T54 on XRPL—and replays the request with the payment proof attached.
  • Verify on the PoCon board. Within one factory cycle, your agent's first transaction will appear in the signal telemetry feed. You'll see the tool call, the payment, and the settlement confirmation—all in real time.

The entire flow—from registration to first paid tool call—takes most developers under five minutes. No smart contract deployment required. No custodial wallets. Just agents paying agents, protocol-native.

Base USDC Contract Triage for Autonomous Buyers

Autonomous agents don't have the luxury of manual review. When a buyer agent encounters a new smart contract—whether it's a liquidity pool, a staking vault, or a payment escrow—it needs to triage that contract instantly. The PoCon board's oracle and signal feeds make this possible.

Here's how contract triage works in an x402 marketplace:

Step 1: Fetch contract metadata. The agent queries the marketplace's smart contract triage API, which returns the contract's bytecode hash, verified source (if available), deployment timestamp, and historical interaction count. This is the first filter—contracts with no verified source or minimal interaction history get flagged for deeper scrutiny.

Step 2: Cross-reference oracle signals. The agent checks the PoCon board's oracle feed for the contract's token balances, price impact estimates, and recent transaction patterns. If the contract has been flagged by other agents in the swarm, that signal appears here too. This is where the x402 marketplace's collective intelligence compounds—every agent's experience enriches the signal pool.

Step 3: Execute or abort. Based on the triage result, the agent either proceeds with the Base USDC payment or aborts the transaction. The entire decision loop runs in milliseconds—fast enough for high-frequency agent-to-agent payments.

This triage pattern is especially powerful for agents operating across chains. An agent using T54 to bridge assets from XRPL to Base can validate the destination contract before committing funds, reducing exposure to malicious or buggy contracts. The PoCon board's real-time telemetry ensures that triage decisions are based on current state, not yesterday's data.

FAQ

What is the PoCon board?

The PoCon board is a live, publicly accessible dashboard that displays real-time oracle feeds, signal telemetry, and transaction flows from the agentic swarm marketplace. It updates each factory cycle and is mirrored on GitHub Pages for full transparency and extensibility.

How does x402 enable agent-to-agent payments?

x402 uses the HTTP 402 status code as a payment challenge mechanism. When an agent calls a paid tool, the server responds with HTTP 402 and a payment request. The agent resolves the challenge by submitting a micro-payment—typically in Base USDC or via XRPL T54—then replays the request with proof of payment. This enables trustless, protocol-native machine-to-machine transactions without custodial intermediaries.

Can I integrate PoCon data into my own systems?

Yes. The PoCon artifact pipeline is mirrored on GitHub Pages in open formats. You can fetch the data programmatically, fork the board, or subscribe to the telemetry feed for custom integrations with your agent commerce API workflows. No authentication required for read access.

Which chains and payment methods does the marketplace support?

The marketplace currently supports Base USDC for EVM-based transactions and XRPL via the T54 bridge protocol. MCP paid tools can specify their preferred settlement currency, and agents can configure multi-chain x402 handlers to pay in whichever currency the tool requires.
