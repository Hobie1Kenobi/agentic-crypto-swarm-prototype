How x402 Works on Base

The x402 protocol extends HTTP with a payment negotiation layer. When an agent requests a paid resource, the server responds with a 402 status and a payment payload describing the cost, recipient, and settlement chain. The agent's client library constructs a USDC transfer on Base, attaches the transaction hash as a payment token, and re-issues the request. The server verifies on-chain settlement and returns the requested data.

This flow is atomic from the agent's perspective: one logical request, one payment, one response. Under the hood, Base's low gas fees and 2-second block times make sub-cent micropayments economically viable. A $0.002 API call costs roughly $0.0001 in gas—well within the margin for autonomous decision-making loops.

Exact-Scheme USDC Micropayments

USDC on Base is an ERC-20 token with 6 decimal places of precision. The x402 exact-scheme leverages this granularity to price resources at fractions of a cent. Contract triage endpoints charge per contract analyzed. Audit data endpoints charge per vulnerability report. Commerce data endpoints charge per signal delivered.

This is not a streaming payment model or a subscription tier. Each API call is an independent economic event. Agents that make 10 calls pay for 10 calls. Agents that make 10,000 calls pay for 10,000 calls. The unit economics are transparent, predictable, and verifiable on-chain.

The XRPL T54 x402 integration means agents aren't locked into a single settlement asset. USDC on Base for EVM-native operations. XRP on XRPL for ledger-native operations. The protocol adapts; the agent stays focused on its objective.

FAQ

What is x402 and how does it differ from traditional API payment?

x402 is an HTTP extension that embeds payment negotiation directly into the request-response cycle. Unlike traditional API keys or subscription models, x402 enables per-call micropayments settled on-chain. The server responds with a 402 status and payment details; the client pays and re-requests. No billing systems, no invoices, no accounts receivable.

Why Base mainnet for USDC agent payments?

Base offers sub-cent transaction fees, 2-second block times, and native USDC support as an official Coinbase L2. These properties make it the most cost-effective chain for high-frequency, low-value micropayments—the exact profile of agent-to-agent API calls. A $0.001 payment on Base costs less than $0.0001 in gas.

How do MCP paid tools work with x402?

MCP (Model Context Protocol) provides tool discovery and invocation. When combined with x402, MCP tools can declare payment requirements in their metadata. Agents discover paid tools through the MCP registry, negotiate payment via x402, and receive results—all without custom payment integration. The marketplace's MCP connect endpoint handles the full lifecycle.

Can agents pay with assets other than USDC?

Yes. While USDC on Base is the primary settlement asset for eip155:8453 operations, the T54 protocol extends x402 to XRPL, where XRP serves as the settlement asset. Agents can use USDC for Base-native calls and XRP for XRPL-native calls, with the same x402 negotiation flow on both chains.
