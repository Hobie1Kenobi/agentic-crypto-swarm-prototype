Open Source Swarm: Build Agent Commerce with x402 and MCP

The agentic crypto swarm prototype is live on GitHub. Connect LangGraph agents, x402 sellers, and MCP paid tools in minutes.

Autonomous agents need more than prompts—they need wallets, protocols, and a marketplace to transact in. Today we're opening the agentic-crypto-swarm-prototype, a fully open-source stack that lets you spin up LangGraph agents, deploy Foundry contracts, expose x402 sellers, and serve MCP paid tools in a single repo. If you've been waiting for a starting point for agent commerce, this is it.

What's Inside the Swarm Prototype

The repository is a batteries-included starter for agentic commerce on Celo, Base, and XRPL. It ships with four core components that work together out of the box:

  • Foundry contracts — Solidity smart contracts for Celo-first deployments, including payment escrows and seller registries. Write once, test locally with Forge, and deploy to Celo mainnet or testnet.
  • LangGraph agents — Configurable agent graphs that reason over on-chain data, call external APIs, and execute transactions. Each agent is a node in a larger swarm, coordinating through shared state and tool calls.
  • x402 sellers — HTTP 402 payment-required endpoints that gate access to data, compute, or AI inference behind a micropayment. Buyers send Base USDC or XRPL T54 tokens; sellers verify on-chain and release the resource. No API keys, no billing portals—just the protocol.
  • MCP server — A Model Context Protocol server that exposes your agent's tools to any MCP-compatible client. Connect agents via MCP and x402 in under five minutes, and you have a machine-to-machine commerce loop running.

Every component is modular. Swap the LLM provider, add a new chain, or extend the agent graph—the architecture is designed for contribution, not configuration lock-in.

Connect Agents via MCP + x402 in Under 5 Minutes

The fastest path to a live agent commerce flow looks like this:

1. Clone and install

Pull the repo, install dependencies, and set your environment variables. The README lists every required key—wallet private key, LLM API key, and RPC endpoints for Celo, Base, and XRPL.

2. Deploy a seller endpoint

Run the x402 seller service. It listens for HTTP requests, checks the incoming payment header, verifies the on-chain transaction, and responds with the gated resource. Base USDC contract triage for autonomous buyers is handled automatically—the seller validates that the USDC transfer landed on Base before releasing data.

3. Register tools on the MCP server

Define your agent's callable functions as MCP tools. The server advertises them to connected clients, including other agents in the swarm. When a buyer agent discovers your tool via the agent commerce API, it can pay and call in a single loop.

4. Run the LangGraph agent

Launch the agent graph. It will discover MCP tools, evaluate which ones to purchase based on task requirements, send x402 payments, and consume the results. The entire cycle—discover, pay, call, reason—happens without human intervention.

From clone to a live agent-to-agent payment, you can be running in under five minutes. That's the power of combining MCP's tool discovery with x402's payment protocol: no accounts, no invoices, just autonomous commerce.

Multi-Chain Payments: Celo, Base, and XRPL

Agent commerce doesn't live on one chain. The swarm prototype is Celo-first for development—cheap gas, fast finality, and a developer-friendly stack—but it's built for inbound machine payments across multiple networks:

  • Base USDC x402 — Accept USDC payments on Base for contract triage, data access, or compute tasks. Base's low fees and Ethereum-secured settlement make it ideal for high-frequency micropayments between agents.
  • XRPL T54 x402 — Accept payments on the XRP Ledger using the T54 token standard. XRPL's sub-second finality is a natural fit for real-time agent-to-agent payments where latency matters.
  • Celo-native deployments — Deploy seller contracts, escrows, and registries on Celo. The Foundry setup targets Celo by default, with configuration overrides for Base and XRPL.

The smart contract triage API within the repo routes incoming payments to the correct verification logic based on the chain and token. Your seller endpoint doesn't need to know whether the buyer paid on Base or XRPL—it just needs to know the payment landed.

This multi-chain approach also positions the swarm as a Coinbase Bazaar seller candidate. The CDP Bazaar connects autonomous buyers with verified sellers; the prototype's x402 endpoints and MCP tools are ready to list.

FAQ

What is x402 and how does it enable agent commerce?

x402 is a protocol that uses the HTTP 402 Payment Required status code to gate access to digital resources. Instead of API keys or subscription billing, a buyer sends an on-chain payment and includes proof in the request header. The seller verifies the payment on-chain and releases the resource. For agents, this means autonomous, trustless transactions—no human in the loop.

How do MCP paid tools work in the swarm?

The Model Context Protocol (MCP) lets agents discover and call tools exposed by other agents or services. In the swarm prototype, each MCP tool can be gated behind an x402 payment. A buyer agent discovers the tool, sends payment, and calls the tool in a single automated flow. This creates a machine-to-machine marketplace where agents buy capabilities from each other.

Which blockchains are supported for payments?

The prototype supports Celo for contract deployments, Base for USDC payments, and XRPL for T54 token payments. The smart contract triage API automatically routes payment verification to the correct chain, so sellers can accept payments from multiple networks without custom logic.

How do I become a Coinbase Bazaar seller?

The Coinbase Developer Platform (CDP) Bazaar connects autonomous buyers with verified sellers. By deploying x402 endpoints and MCP tools from the swarm prototype, your services are structurally ready for Bazaar listing. Check the CDP documentation for the latest seller onboarding process.
