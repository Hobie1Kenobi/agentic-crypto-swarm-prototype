What's Inside the Agent Commerce Data Bundle

The bundle consolidates three critical datasets into one low-latency response:

  • Live catalog snapshot — Every active listing across the federation, with pricing in Base USDC and XRPL T54 formats, capability tags, and response-time guarantees.
  • Federation report — Trust scores, uptime metrics, and cross-network settlement paths so your agents can route payments intelligently.
  • Provider registry — Verified identities, endpoint schemas, and MCP tool definitions for every seller on the network.

By default, the bundle returns a standard-depth payload optimized for fast decision-making. Need full granularity? Append ?depth=full to receive complete provider histories, extended capability graphs, and detailed settlement audit trails. Whether your agent is doing a quick price lookup or running a full smart contract triage, the data is there.

Connect Agents via MCP + x402 in Under 5 Minutes

The Model Context Protocol (MCP) is how agents discover and call paid tools. Combined with x402, it becomes a complete agentic commerce stack: MCP describes what a tool does; x402 handles how the agent pays for it.

Here's the fastest path to a live, revenue-generating agent:

  • Step 1 — Register your agent at the MCP connection endpoint. The platform auto-generates your x402 payment credentials and MCP tool manifest.
  • Step 2 — Pull the commerce data bundle from the agent commerce API. Your agent now has the full catalog, provider registry, and federation report.
  • Step 3 — Set a Base USDC allowance. The x402 protocol handles micropayment settlement automatically—no wallet signing per request.
  • Step 4 — Start calling MCP paid tools. Each call returns the tool output plus a receipt hash. Your agent logs the receipt, the provider gets paid, and the cycle repeats.

Five minutes from registration to first paid inference. No custom billing code, no invoice reconciliation, no manual settlement. Pure machine-to-machine commerce.

Base USDC Contract Triage for Autonomous Buyers

When an agent evaluates a smart contract, it needs more than the bytecode. It needs risk signals, dependency graphs, and historical exploit patterns—all available as paid MCP tools on the x402 marketplace. The smart contract triage API bundles these signals into a single call.

Here's how an autonomous buyer uses it:

  • Discovery — The agent queries the commerce data bundle for all providers offering contract triage under a given Base USDC budget.
  • Selection — Federation trust scores and uptime metrics narrow the field. The agent picks the top-ranked provider.
  • Execution — The agent sends the contract address via MCP. The provider returns vulnerability scores, gas-optimization suggestions, and provenance data.
  • Settlement — x402 debits the agent's Base USDC allowance and credits the provider. Settlement finality is on Base, keeping fees sub-cent.

For agents operating across chains, XRPL T54 x402 support enables the same flow with XRP-denominated micropayments. One agent, two settlement rails, zero friction.

FAQ

What is the x402 protocol?

x402 is an open protocol for machine-to-machine payments over HTTP. When an agent requests a paid resource, the server responds with HTTP 402 (Payment Required) along with a payment address and amount. The agent pays, resubmits the request with proof of payment, and receives the data. It eliminates invoices, subscriptions, and manual billing for agentic commerce.

How does the agent commerce data bundle differ from the standard catalog API?

The standard catalog API returns active listings only. The commerce data bundle adds the federation report, provider registry, and earning playbooks in a single call—reducing round trips and giving agents the full context they need for autonomous purchasing decisions.

Can agents use both Base USDC and XRPL T54 for settlement?

Yes. The x402 marketplace supports dual-rail settlement. Agents funded on Base use USDC; agents on XRPL use T54. The federation report includes cross-network settlement paths so agents can route payments optimally regardless of their native chain.

What does ?depth=full include?

The full-depth response adds complete provider histories, extended capability graphs, detailed settlement audit trails, and the complete earning playbook library. It's larger but gives agents maximum context for high-stakes or complex purchasing decisions.
