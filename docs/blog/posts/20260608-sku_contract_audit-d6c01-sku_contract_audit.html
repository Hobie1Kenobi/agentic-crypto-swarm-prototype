The End of Manual Audit Bottlenecks

Traditional smart contract audits are designed for human consumption. They take days, cost thousands of dollars, and output static documents. While essential for major protocol launches, this model fundamentally breaks down in the context of agent commerce. When an agent discovers a new yield vault or a liquidity pool, it needs to assess risk instantly.

Our Full Contract Audit endpoint replaces the manual bottleneck with automated, deterministic analysis. The system runs a comprehensive pipeline that includes:

  • Slither Static Analysis: Rapidly scans the contract's Solidity code for common vulnerabilities, uninitialized storage pointers, and unchecked external calls. Slither provides the fast, high-level triage required to filter out obviously malicious or flawed contracts.
  • Echidna Property-Based Fuzzing: Goes beyond static patterns by actively fuzzing the contract's invariants. Echidna generates arbitrary calldata to break defined properties, uncovering deep logic errors and edge cases that static analysis misses.
  • Multi-Address Scanning: Evaluates up to 3 addresses per request, allowing agents to analyze complex multi-contract interactions (e.g., a proxy, an implementation, and a reward token) in a single atomic request.

This dual-pronged approach ensures that agents are protected from both well-known attack vectors and complex, protocol-specific exploits.

Deep On-Chain Intel: Deployers, Delegates, and Concentration

Secure code does not guarantee a secure investment. A flawless contract deployed by a known bad actor or heavily concentrated in a few wallets remains a massive risk. Our audit endpoint integrates deep on-chain intelligence to provide the full context.

Deployer Profiling

We trace the deploying address across its entire history. Has this deployer launched rugs before? Are the admin functions controlled by a multi-sig or a single point of failure? Agents need to know who is steering the ship before boarding.

EIP-7702 Delegate Detection

With the advent of EIP-7702, Externally Owned Accounts (EOAs) can temporarily delegate their code to a contract. This fundamentally shifts the security model of Base and Ethereum. Our audit endpoint specifically scans for EIP-7702 delegate designations, alerting agents to delegated code contexts that might alter execution flow or introduce unexpected attack surfaces.

Holder Concentration Analysis

We calculate the distribution of token holders. A token where 90% of the supply is held by three wallets is a trap, regardless of how well-audited the code is. Our endpoint flags high holder concentration, allowing agents to automatically adjust slippage tolerances or reject the interaction entirely.

Structured Output for Agent Pipelines

Agents do not read PDFs. They parse JSON. The ultimate deliverable of our Full Contract Audit is the audit_report_v2 JSON payload. This schema is explicitly designed for programmatic consumption by autonomous agents.

Instead of parsing ambiguous text, your agent receives a structured response detailing:

  • Severity Scores: Clear, numerical risk ratings for immediate go/no-go decisions.
  • Findings: Categorized arrays of vulnerabilities detected by Slither and Echidna.
  • On-Chain Metrics: Deployer reputation scores, EIP-7702 delegate flags, and holder concentration percentages.

This structured output plugs directly into your agent's decision-making pipeline. If the holder_concentration metric exceeds your risk threshold, the agent aborts. If Echidna breaks an invariant, the agent halts. It is security by code, not by committee.
